Logging Screen
Screen Heading: Settings > Logging
Use the Logging screen to view the list of loggers that Integration Server employs to record logging information about different facilities within Integration Server.
Important!  
The Guaranteed Delivery logging and Security logging functions are available under separate licenses. If you are not licensed to use a given logger, the Enabled field for this logger will display No and you will not be allowed to enable it from this screen.
To open the Logging screen, click Logging under the Settings menu on the Navigation panel.
To view the screen for a particular logger, click the logger name.
Logger List
The logger list displays the following information about the different loggers:
Field
Description
Logger Name
Name of the logger associated with a given facility.
Enabled
Whether or not Integration Server is to write log entries for this facility. The default is Yes for all loggers except the security logger.
Mode
Mode for writing log entries.
Synchronous
The logger writes log entries directly to the destination.
Asynchronous
The logger writes log entries to a queue, and the queue later writes the entries to the destination. This is the default.
Guaranteed
How Integration Server is to maintain the queue when the logger is logging asynchronously.
No
Integration Server maintains the queue in memory. This is the default.
Yes
Integration Server persists the queue on disk.
Destination
Destination for log entries.
Database
The logger writes log entries to an external RDBMS. You must also set the Maximum Retries and Wait Between Retries fields.
This value is available when the IS Core Audit Log connects to an external RDBMS at Integration Server startup.
File
The logger writes log entries to file.
