Session Log Screen
Screen Heading: Logs > Session
Use the Session Log screen to view information in the server's Session Log. The Session Log (WMSESSION_yyyymmdd_hhmmss.log) maintains information about sessions that are established with the webMethods Integration Server. The webMethods Integration Server creates a session when a developer connects to the server with Software AG Designer or a webMethods client connects to the server to execute services. Each record in the Session Log includes a timestamp indicating when a user established a connection with the webMethods Integration Server. For information about changing the display of this log, see webMethods Audit Logging Guide .
To open the Session Log screen, click Session under the Logs menu on the Navigation panel.
Log display controls
Display Log Entries oldest to the newest starting from the beginning
View entries in the log from the oldest to the most current.
Display Log Entries newest to the oldest starting from the end
View entries in the log from the most current to the oldest.
Number of entries to display
By default, the Integration Server Administrator displays the 35 most recent entries in the log. You can change the number of entries displayed.
Tip!  
If you have Administrator privileges, you can permanently change the default value of this field. Go to the Settings > Extended page and modify the watt.server.log.maxEntries parameter.
From/To
Specify, in YYYY-MM-DD HH:MM:SS date format, the range of dates and times for which you want log entries to be displayed.
Note:  
The From/To fields are displayed only if the logger associated with this log is configured to write to a database. For more information, see the webMethods Audit Logging Guide .
Refresh
Click this button to update the display after changing the log display controls.
Tip!  
If you have Administrator privileges, you can change the default refresh interval for log entries (the default is 90 seconds). Go to the Settings > Extended page and modify the watt.server.log.refreshInterval.
Session Log Entries as of mm-dd hh:mm:ss
Time Stamp
Date and time the entry was written to the log.
Server Id
Identity of the server on which the audit event occurred. The ID can be DNSname:port or ipaddress:port.
User Id
Identity of the user that is connected for this session.
Client IP
IP address of the client machine. The word "system" appears for session requests from Integration Server for operations such as running a scheduled service or refreshing the display.
Session State
Possible values are STARTED, EXPIRED, and ENDED.
RPCs
Number of times the service has been invoked this session.
Age
How long the session existed (in milliseconds) before it expired or ended.
Session ID
A string the server generates to identify each session uniquely.
