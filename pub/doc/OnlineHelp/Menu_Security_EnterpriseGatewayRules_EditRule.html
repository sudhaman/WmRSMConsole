Edit Rule Screen
Screen Heading: Security > Enterprise Gateway Rules > Rules > rulename > Edit
Use the Edit Rule screen to view or change rule properties.
To open the Edit Rule screen, click Enterprise Gateway Rules under the Security menu, and then click the rule name.
Click Return to Enterprise Gateway Rules to return to the Enterprise Gateway Rules screen.
Rule Properties
Rule Name
Name of the rule.
Valid rule names:
Must be unique.
Must not be null.
Must not contain spaces.
Must not contain these special characters:
? ~ ` ! @ # $ % ^ & * ( ) - + = { } | [ ] \\ : \" ; ' < > , /
Description
Description of the rule.
Enable
Whether to enable or disable the rule.
Request Type
The type of request to which you want to apply the rule. Select ALL if you want to apply this rule to all requests. Select SOAP, REST, or INVOKE to apply this rule to SOAP, REST, or INVOKE requests respectively.
Resource Path
A path to the resource being requested. Specify a string in the format folder_name/service_name. You can specify multiple paths, one to a line, and you can specify an asterisk (*) wildcard character. The Resource Path field is available only if you selected a request type from the Request Type list.
Action
One of the following:
Select...
To...
Deny Request and Alert
Deny the request if it violates the rule and send an alert based on the configured alert options.
Alert
Allow the request and send an alert based on the configured alert options.
Error Message
Custom message that you want Enterprise Gateway Server to send to the client if a request violates an Enterprise Gateway deny rule. This message is sent as a status message for an HTTP 403 status code. For example, the client will receive “HTTP 403 custom error message.”
OAuth Filter
Enable
Whether Enterprise Gateway Server should require requests to contain an OAuth token.
Require OAuth Credentials
Whether Enterprise Gateway Server requires requests to contain an OAuth token in the authorization header of the request.
Message Size Filter
Enable
Whether Enterprise Gateway Server is to check the message size.
Maximum Message Size
Maximum size allowed for HTTP and HTTPS requests, in megabytes. If the request is larger than the size specified in this limit, the rule is violated and Enterprise Gateway Server performs the configured action.
Mobile Application Protection Filter
Enable
Whether Enterprise Gateway Server is to check the requests from mobile applications for application name and version, and the type of device from which the request was sent. When a request meets the condition specified in the filter, depending on how the rule is configured, Enterprise Gateway Server will either deny the request and send an alert or just send an alert.
Device Type
Type of mobile device from which the request was sent.
Mobile Application
Name of the mobile application that sent the request.
Condition
An operator that helps define which versions of the application you want to deny or send an alert about. Can be one the following:
Symbol
Meaning
=
Equal to
>
Greater than
<
Less than
>=
Greater than or equal to
<=
Less than or equal to
<>
Not equal to
Mobile Application Version
Mobile application version that you want to check for. In combination with the condition operator, the version allows you to specify a variety of conditions. For example, you can deny all requests from an unsupported version of an application, or you can send an alert about all requests received from the most current version of the application.
Add
Adds another row into which you can enter more conditions.
Delete
Deletes a row of conditions.
Save Changes
Saves the Enterprise Gateway rule.
Enterprise Gateway Server displays the rule in the Enterprise Gateway Rules screen under Denial Rules or Alert Rules depending on the action you selected when you created the rule.
