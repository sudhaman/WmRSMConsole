OAuth Screen
Screen Heading: Security > OAuth
Use the OAuth screen to view the current settings for the application server.
To open the OAuth screen, click OAuth under the Security menu on the Navigation panel.
Click Client Registration to configure a new client.
Click Scope Management to add and delete scopes, and to associate scopes to clients.
Click Tokens to view and delete active tokens.
Click Edit OAuth Global Settings to edit the authorization server settings.
Authorization Server Settings
Require HTTPS
Indicates whether the authorization server should require an HTTPS connection to authorize requests. If enabled (the default), Integration Server requires that the authorization server uses HTTPS to invoke the pub.oauth:authorize, pub.oauth:getAccessToken, and pub.oauth:refreshAccessToken services.
Important!  
If Require HTTPS is disabled, Integration Server will issue an exception.
Authorization code expiration interval
Specifies the length of time (in seconds) that the authorization code issued by the authorization server is valid.
Access token expiration interval
Specifies the length of time that access tokens issued by the authorization server are valid. Never Expires indicates that the access token never expires.
Resource Server Settings
Authorization server
Indicates the Integration Server to use as the authorization server.
