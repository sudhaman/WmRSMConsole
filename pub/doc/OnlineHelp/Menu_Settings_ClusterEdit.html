Edit Cluster Settings Screen
Screen Heading: Settings > Cluster > Edit
Use the Edit Cluster Settings screen to edit settings that control how your server operates when it is part of a cluster. For more information about clustering, see the webMethods Integration Server Clustering Guide .
To open the Edit Cluster Settings screen, go to the Clustering Screen and click Edit Cluster Settings.
Click Return to Cluster Settings to return to the list of current cluster settings.
Click Enable Cluster to add this server to a cluster.
Click Disable Cluster to disengage this server from a cluster.
General
Use the fields in this section to specify general information about the cluster.
Clustering Status
Indicates whether the server is a member of a cluster.
Cluster Name
Name of the cluster to which this Integration Server belongs. If you specify a name that does not already exist, the caching software creates a new cache for this cluster. If you specify the name of a cluster that already exists, the caching software adds this Integration Server to that cluster. An enterprise can have more than one cluster. The cluster name allows the caching software to form separate caches for each cluster. Without a cluster name, all Integration Servers that share the same Discovery Address and Discovery Port would form a single cache.
Session Timeout
Number of minutes an inactive session will be retained in the clustered session store. Default is 60.
Set this timeout value longer than the session timeout value, which governs how long a server keeps session information in its memory. (The server displays the session timeout on the Resources screen.)
Terracotta Server Array URLs
A comma separated list of the URLs for the Terracotta Server Array that contain the distributed system caches for the cluster.
Cluster Hosts
This section displays information about other Integration Servers in the cluster.
Name
Domain and host name of the machine on which the Integration Server runs.
Address
IP address of the machine on which the Integration Server runs.
Server Start Time
Time the Integration Server started.
Server Up Time
Length of time (hh:mm:ss) that the server has been running since it was last started.
Total Memory
Total amount of memory available to the Java virtual machine.
Free Memory
Amount of unused memory that is currently available to the Java virtual machine.
Used Memory
Amount of memory currently used by the Java virtual machine.
Current Server Threads
Number of running services currently active.
Max Server Threads
Maximum number of threads that the server maintains in the server thread pool. If this maximum number is reached, the server waits until processes complete and return threads to the pool before starting more processes. The default is 75.
Current System Threads
Number of system threads currently active. Click this number to display the Threads screen, which contains detailed information about the threads that are currently active.
Max System Threads
Highest number of system threads that have ever been active concurrently on the server.
Total Requests
Total number of requests that were completed during the polling interval (the last 10 minutes on most servers).
Average Duration
Average time to complete a request.
