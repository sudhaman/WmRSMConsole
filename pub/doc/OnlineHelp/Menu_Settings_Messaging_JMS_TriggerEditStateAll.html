Edit State for All JMS Triggers Screen
Screen Heading: Settings > Messaging > JMS Trigger Management > Edit State
Use the Edit State for All JMS Triggers screen to enable, disable, suspend, or for all JMS triggers.
To open the Edit State for All JMS Triggers screen, click Messaging under the Settings menu on the Navigation panel. Under JMS Configuration on the Messaging Screen, select JMS Trigger Management. In the Individual JMS Trigger Controls section, there is a heading named Enabled. To edit the state of all enabled JMS triggers, click the edit all link under Enabled.
Click Return to JMS Trigger Management to return to the previous screen.
Edit Triggers State
Use this section to change the state for all the JMS triggers on the Integration Server.
New State
Specifies the state for the JMS triggers on the Integration Server.
Select...
To...
Enabled
The JMS trigger is running and connected to the JMS provider. Integration Server retrieves and processes messages for the JMS trigger.
Disabled
The JMS trigger is stopped. Integration Server neither retrieves nor processes messages for the JMS trigger.
Suspended
The JMS trigger is running and connected to the JMS provider. Integration Server has stopped message retrieval, but continues processing any messages it has already retrieved.
JMS Trigger Type
Specifies whether the state change applies to all the JMS trigger on Integration Server or a subset of the JMS triggers.
Select...
To...
All
Apply the state change to all JMS triggers.
Standard
Apply the state change to standard JMS triggers only.
SOAP
Apply the state change to SOAP JMS triggers only.
Save Changes
Saves the specified settings. Server restart is not required for these changes to take effect. Any changes you make are persisted across server restarts.
