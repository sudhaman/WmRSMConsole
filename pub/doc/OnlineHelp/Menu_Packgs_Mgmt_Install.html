Install Inbound Releases Screen
Screen Heading: Packages > Management > Install Inbound Releases
Use the Install Inbound Releases screen to install packages that other webMethods Integration Servers have published to your server. These reside in distribution files (zip files) in the Integration Server_directory \instances\instance_name\replicate\inbound directory on your server.
To open the Install Inbound Releases screen, open the Packages Management Screen and click Install Inbound Releases.
Click Return to Package Management to return to the main package management screen.
Release file name
Lists the distribution files that are available for installation.
Option-Activate upon installation
Specifies whether you want the server to activate the package immediately upon installation. If you want the package to be available to clients immediately after you install it, enable this option. If you want to install the package, but activate it at a later time, disable this option. (If you do not activate the package upon installation, use the Activate Inactive Packages Screen to activate it manually when you are ready to make it available to users.)
Option-Archive upon installation
Specifies whether you want Integration Server to archive the package automatically after installation. To archive the package automatically after installation, select the Archive upon installation check box. This check box is selected by default.
If you choose to archive the package automatically, Integration Server appends a timestamp to the filename and moves the package from the Integration Server_directory \instances\instance_name\replicate\inbound directory to the Integration Server_directory \instances\instance_name\replicate\archive directory.
If you do not want to archive the package automatically after installation, clear this check box. To archive the package later, go to the Archive Package Screen.
Install Release
Installs the distribution file specified in Release File Name. If you install a package that already exists on your server, your server copies the original package to Integration Server_directory \instances\instance_name\replicate\salvage before it installs the new one. This lets you revert easily to the previous version if you are dissatisfied with the new package. For information about reverting to the earlier version, see the Recover Packages Screen.
