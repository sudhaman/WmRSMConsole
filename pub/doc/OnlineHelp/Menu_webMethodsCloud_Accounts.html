Accounts Screen
Screen Heading:webMethods Cloud > Accounts
Use the Accounts screen to view, test, enable, disable, and delete webMethods Cloud accounts.
To open the Accounts screen, click Accounts under the webMethods Cloud menu on the Navigation pane.
Click Create On-Premise Account to create and test webMethods Cloud accounts.
On-Premise Accounts
This section lists the accounts that exist on the on-premise Integration Server.

Test
Validates the account. To test the account settings, click the icon in the Test column for the account you want to test. Integration Server Administrator displays a status line that indicates whether the account is valid. The status line is displayed at the top of the screen.
Enabled
Displays the status of the account. When the account is enabled, the on-premise Integration Server automatically establishes connectivity with webMethods Cloud at startup. You can block incoming requests on an account by disabling it. When an account is disabled, messages sent to the on-premise Integration Server remain in the queue until they are read or expire. To disable an account, click the Yes link in this column. To re-enable an account, click the No link in this column.
Note:  
Integration Server does not enable webMethods Cloud accounts by default. After you create an account, you must manually enable the account before you can use it.
Delete
Deletes the account specified in Alias. To delete an account, click the icon in the account’s Delete column.
Note:  
When you delete an account from an on-premise Integration Server, the account is also deleted from the webMethods Cloud.
