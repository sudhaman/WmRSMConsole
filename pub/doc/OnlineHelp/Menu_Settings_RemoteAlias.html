Create Remote Server Alias Screen
Screen Heading: Settings > Remote Servers > Create Alias
Use the Create Remote Server Alias screen to define an alias for a remote webMethods Integration Server. An alias is a named set of parameters that describes the way in which your server connects to another webMethods Integration Server.
To open this screen, open the Remote Servers Screen and click Create Remote Server Alias.
Click Return to Remote Servers to return to the list of aliases defined on your server.
Alias
Specifies the name of the alias. You can give the remote server any alias name but it cannot include the following illegal characters: # - & @ ^ ! % * : $ . / \ \ ` ; , ~ + = ) ( | } { ] [ \> < " .
Host Name or IP Address
Specifies the domain name or numeric IP address of the remote server.
Port Number
Specifies the port number on the remote server to which requests will be sent when this alias is used.
User Name
Specifies the user name that your server will use when it submits requests using this alias. Make sure to specify a user name that has access to the resources that your server will need to use on the remote server.
Password
Specifies the password for the user account specified in User Name.
Execute ACL
Specifies the ACL that controls who can use this alias. Select an ACL from the drop down list.
Max Keep Alive Connection
Sets the default number of client keep alive connections to retain for a given target endpoint. If not specified, five keep alive connections are retained. This field specifies the maximum number of client connections that should be retained for any given remote host. In other words, this is not a maximum number of connections that can be established, but a limit on the number of inactive client connections to retain for reuse.
Keep Alive Timeout (minutes)
Displays the length of time (in minutes) that your server maintains an idle connection to a remote server. This value will cause the connection to be retained for possible reuse until it times out. If the specified keep alive timeout value expires, the connection will close and the HTTP Listener will attempt to create a new one.
Use SSL
Specifies whether you want your server to connect to the remote server using Secure Sockets Layer (SSL).
Note:  
If you select Yes for this parameter, the remote server must be configured to listen for incoming HTTPS requests on the port specified in Port Number.
Keystore Alias (optional)
Indicates the user-specified identifier for the keystore containing the private keys and certificates used for authentication. Integration Server displays this field only if you specify Yes for Use SSL.
Key Alias (optional)
Indicates a user-specified text identifier for a private key and associated certificate chain. Integration Server displays this field only if you specify Yes for Use SSL.
Retry Server
Host name or IP address (e.g., workstation6.webmethods.com) of a remote server you want your local Integration Server to connect to if the primary remote server is unavailable. Specify this parameter if you are using the remote:invoke and pub.remote.gd:* services to invoke services on a remote server that is part of a cluster.
Save Changes
Adds the alias to the server's alias list.
