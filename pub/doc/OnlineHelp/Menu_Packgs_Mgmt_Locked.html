View Locked Elements Screen
Related Topics
Screen Heading: Packages > Management > Locked Elements
Use the View Locked Elements screen to show a list of user and system locked elements on the webMethods Integration Server and unlock elements.
To open the View Locked Elements screen, open the Packages Management Screen and click View Locked Elements.
Click Return to Package Management to return to the main package management screen.
Click Unlock Elements to unlock elements that are locked by you or other users on the Integration Server. Make sure that you inform users before removing their locks.
Click Sync to Name Space if there are locked elements shown that do not have corresponding files on the Integration Server (in other words, the files were deleted from the server). This command updates the Integration Server Administrator's view of unlocked elements by polling the Integration Server and updating the list. Note that this command is automatically run when the Integration Server is started or restarted.
Locked by System
Lists elements whose corresponding files are marked read-only on the Integration Server file system. You cannot remove a system lock via the Integration Server Administrator. On the server’s file system, you must make the element’s files writable and reload the package.
Locked by Current User (username)
Lists elements that are locked by you, the server administrator (or the username with which you logged into the Integration Server Administrator). Before you unlock an item, make sure that you have saved all changes to the Server.
Locked by Other Users
Lists elements that are locked by other users on the server. Before you remove a user’s lock, make sure that the user has saved all changes to the server. If not, the user will lose all changes that they made to the element since they last saved it to the server.
Lock Details Screen
Screen Heading: Packages > Management > Locked Elements > Lock Details
Use the Lock Details screen to view detailed information about a locked element on the Server. To open this screen, click one of the elements listed on the View Locked Elements Screen.
To open the Lock Details screen, open the Packages Management Screen and click View Locked Elements. Click the name of the element to view the details of the lock.
Click Return to Locked Elements to return to the list of locked elements on the Server.
Name
Displays the fully-qualified name of the element.
Lock Status
Displays the user name of the person who currently owns the lock on the element.
Host
Displays the address of the Server on which the element resides.
Lock Time
Displays the time at which the element was initially locked.
Related Locks
Displays the fully-qualified names of elements that share the same lock within the folder. For example, if the element you locked is a Java service, all Java services within the folder will be locked and displayed in this field.
Unlock Elements Screen
Screen Heading: Packages > Management > Unlock Elements
Use the Unlock Elements screen to remove user locks from elements on the Integration Server.
To open the Unlock Elements screen, open the Packages Management Screen and click View Locked Elements. On the View Locked Elements Screen, click Unlock Elements.
Important! Exercise caution when you remove user locks, since a user can lose changes. If you unlock an element while a user is editing it, the user will not know until he or she tries to save changes to the element and the operation fails. Always confirm with the user before removing his or her lock on an element.
Click Return to Locked Elements to return to the View Locked Elements Screen.
Locked by System
Lists elements whose corresponding files are marked read-only on the server file system. You cannot remove a system lock via the Integration Server Administrator. On the server’s file system, you must make the element’s files writable and reload the package.
Locked by Current User (username)
Lists elements that are locked by you, the server administrator (or the username with which you logged into the Integration Server Administrator). Before you unlock an item, make sure that you have saved all changes to the server.
Locked by Other Users
Lists elements that are locked by other users on the server. Before you remove a user’s lock, make sure that the user has saved all changes to the server. If not, the user will lose all changes that they made to the element since they last saved it to the server.
Unlock Selected Elements
Click to unlock the elements with selected checkboxes in the list. Make sure that you inform users before removing their locks.
Note: If you receive an “element failed to unlock” message after clicking Unlock Selected Elements, then you may have elements listed that do not have corresponding files on the Integration Server (in other words, the files were deleted from the server). To update the view of locked elements in the Integration Server Administrator, use the Sync to Name Space command on the previous screen.
