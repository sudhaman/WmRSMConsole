JNDI Provider Alias Details Screen
Screen Heading: Settings > Messaging > JNDI Settings > JNDI Provider Alias
Use the JNDI Provider Alias Management screen to view the configuration data for a JNDI provider alias on the Integration Server.
To open the JNDI Provider Alias Management screen, click Messaging under the Settings menu on the Navigation panel. Under JMS Configuration on the Messaging Screen, select JNDI Settings. In the JNDI Provider Alias Definitions table, under the JNDI Alias Name heading, click the name of the alias that you are interested in.
Click Return to JNDI Settings to return to the previous screen.
Click Edit JNDI Provider Alias to edit the configuration details for the JNDI provider alias.
JNDI Provider Alias
JNDI Alias Name
Displays the JNDI alias name.
Description
Displays the description for this JNDI alias.
Initial Context Factory
Displays the class name of the JNDI provider. The initial context is the starting point that the JNDI provider users to resolve names for naming and directory operations.
Provider URL
Displays the URL of the initial context for sessions with the JNDI provider. The URL specifies the JNDI directory in which the JNDI provider stores JMS administered objects.
Security Principal
Displays the name of the security principal that the JMS administrator provides to the JNDI provider.
Security Credentials
Displays the security credentials that the JMS administrator provides to the JNDI provider.
Other Properties
Displays the additional properties for JNDI configuration required by the JNDI provider.
