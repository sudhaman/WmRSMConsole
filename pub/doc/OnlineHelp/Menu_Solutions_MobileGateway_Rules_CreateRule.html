Create Rule Screen
Screen Heading: Mobile Gateway > Rules > Create
Use the Create Rule screen to create and configure Mobile Gateway rules. You can specify the type of request to which a rule is applied, filters to be applied against a request, and the action Integration Server is to perform when a request violates a rule.
To open the Create Rule screen, click Mobile Gateway under the Solutions menu, click Rules, then click Create Rule.
Click Return to Rules to return to the Rules screen.
Rule Properties
Properties of the rule.
Rule Name
Name for the Mobile Gateway rule. Valid rule names:
Must be unique.
Must not be null.
Must not contain spaces.
Must not contain these special characters:
? ~ ` ! @ # $ % ^ & * ( ) - + = { } | [ ] \\ : \" ; ' < > , /
Description
Brief description of the Mobile Gateway rule.
Enable
Whether you want to enable the rule immediately. Integration Server applies a rule to requests only if the rule is enabled. You can also enable the rule from the Mobile Gateway > Rules screen.
Applies to
Element to which Integration Server is to apply the rule.
Select...
To...
Request Type
Specify the type of request to which you want to apply the rule. Select All if you want to apply this rule to all requests. Select SOAP, REST, or INVOKE to apply this rule to SOAP, REST, or INVOKE requests respectively.
Resource Path
Specify a path to the resource being requested. Specify a string in the format folder_name/service_name. You can specify multiple paths, one to a line, and you can specify an asterisk (*) wildcard character. The Resource Path field is available only if you selected a request type from the Request Type list.
Action
One of the following:
Select...
To...
Deny Request and Alert
Deny the request if it violates the rule and send an alert based on the configured alert options.
Alert
Allow the request and send an alert based on the configured alert options.
Error Message
Custom message to send to the client when a request violates a Mobile Gateway rule. Integration Server sends an error message only if a request violates a Deny Request and Alert rule. This message is sent as a status message for an HTTP 403 status code. For example, the client will receive “HTTP 403 custom error message.”
OAuth Filter
Enable
Whether the OAuth filter is enabled.
Require OAuth Credentials
Whether Integration Server requires requests to contain an OAuth token in the authorization header of the request.
Message Size Filter
Enable
Whether Integration Server is to check the size of the incoming message.
Maximum Message Size
Maximum size allowed for HTTP and HTTPS requests. If the request is larger than the size specified in this limit, the rule is violated and Integration Server performs the configured action.
Mobile App Protection Filter
Enable
Whether Integration Server is to check the requests from mobile applications for application name and version, and the type of device from which the request was sent. When a request meets the condition specified in the filter, Integration Server will, depending on how the rule is configured, either deny the request and send an alert, or just send an alert.
Device Type
Type of mobile device from which the request was sent.
Mobile Application
Name of the mobile application that sent the request.
Condition
An operator that helps define which versions of the mobile application you want to deny or send an alert about. Can be one the following:
Symbol
Meaning
=
Equal to
>
Greater than
<

Greater than or equal to

Mobile Application Version
Mobile application version that you want to check for. The version you specify, in combination with the condition operator, allows you to specify a variety of conditions. For example, you can deny all requests from an unsupported version of an application, or you can send an alert about all requests received from the most current version of the application. The version must use this format:
major-version.[minor-version.[sub-minor-version.[patch]]]
Add
Adds another row into which you can enter more conditions.
Delete
Deletes a row of conditions.
Save Changes
Saves the Mobile Gateway rule.
Integration Server displays the rule in the Rules screen under Denial Rules or Alert Rules depending on the action you selected when you created the rule.
