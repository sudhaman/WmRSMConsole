Remote Servers Screen
Screen Heading: Settings > Remote Servers
Use the Remote Servers screen to view the list of aliases that your server defines for other webMethods Integration Servers. An alias is a named set of connection parameters that describe how to access another webMethods Integration Server. You use an alias when you invoke a service through the pub.remote:invoke and pub.remote.gd:* services, when you use multiple certificates, and when you use the publish/subscribe features to replicate packages between servers.
To open the Remote Servers screen, click Remote Servers under the Settings menu on the Navigation panel.
Click Create Remote Server Alias to create a new alias.
Remote Servers List
Displays the aliases that are defined on your server.
Alias
Displays the alias name.
Host
Displays the host name or IP address of the remote server represented by this alias.
Port
Displays the port number that is used by the remote server specified by this alias.
User
Displays the user name that is used when a service on the remote server specified by this alias is invoked.
SSL
Whether you want your server to connect to the remote server using Secure Sockets Layer (SSL). If you want to use SSL, select yes; otherwise, select no. If you select yes, the remote server must be configured to listen for incoming HTTPS requests.
Execute ACL
Displays the name of the ACL on your server that protects access to this alias. A client that attempts to invoke a service on the remote server must be authorized to use the alias in addition to the remote service. Choose the name of an ACL from the drop-down menu which provides the following choices: Administrator, Anonymous, Default, Developer, Internal, Replicator.
Keep Alive Conns
Sets the default number of client keep alive connections to retain for a given target endpoint. If not specified, five keep alive connections are retained. This field specifies the maximum number of client connections that should be retained for any given remote host. In other words, this is not a maximum number of connections that can be established, but a limit on the number of inactive client connections to retain for reuse.
Timeout
Displays the length of time (in minutes) that your server maintains an idle connection to a remote server. This value will cause the connection to be retained for possible reuse until it times out. If the specified keep alive timeout value expires, the connection will close and the HTTP Listener will attempt to create a new one.
Test
Tests the remote server alias connection as it is configured.
Delete
Deletes the remote server alias from Integration Server.
