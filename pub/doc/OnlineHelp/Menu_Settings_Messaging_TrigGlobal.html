Edit Global webMethods Messaging Trigger Controls Screen
Screen Heading: Settings > Messaging > webMethods Messaging Trigger Management > Global webMethods Messaging Trigger Controls > Edit
To open the webMethods Messaging Trigger Management screen, click Messaging under the Settings menu on the Navigation panel.
On the Messaging Screen, click webMethods Messaging Trigger Management. On the webMethods Messaging Trigger Management Screen, click Edit Global webMethods Messaging Trigger Controls.
Click Return to webMethods Messaging Trigger Management to return to the previous screen.
Document Retrieval
Maximum Threads
Maximum percentage of the server thread pool that can be used to retrieve documents from the Broker. If the Maximum Threads field under Document Retrieval displays "(Broker Not Configured)", then this Integration Server is not configured to connect to a Broker.
Queue Capacity Throttle
Percentage of configured capacity at which you want all trigger queues to operate. For webMethods messaging triggers that receive documents from the Broker, Integration Server automatically adjusts the refill levels by the same percentage. Refill level does not apply to webMethods messaging trigger that receive documents from Universal Messaging.
Document Processing
Maximum Threads
Maximum percentage of the server thread pool that can be used to process documents received by webMethods messaging triggers. You must enter a value greater than zero. The default is 100%.
Execution Threads Throttle
Percentage of the configured maximum execution threads value at which you want to the server to function. Integration Server applies this percentage to the maximum execution threads value for all concurrent webMethods messaging triggers.
Change Mode
Apply Change Across Cluster
When selected, indicates whether Integration Server applies the global trigger control changes to all the servers in a cluster. This check box appears only if the current Integration Server belongs to a properly configured cluster and is configured to synchronize trigger changes across the cluster.
Save Changes
Saves the specified settings. Integration Server restart is not required for these changes to take effect. Any changes you make are persisted across server restarts.
