WS Endpoint Trigger Details Screen
Screen Heading: Settings > Messaging > JMS Trigger Management > Detail > triggerName
Use the WS (web service) Endpoint Trigger Details screen to view subscription, property values, and current usage information for a WS endpoint trigger.
To open the WS Endpoint Trigger Detail screen, click Messaging under the Settings menu on the Navigation panel. Under JMS Configuration on the Messaging Screen, select JMS Trigger Management. In the Individual SOAP JMS Trigger Controls section, click the name of the WS endpoint trigger for which you want to view details. The Detail screen for that particular JMS trigger will be displayed.
Click Return to JMS Trigger Management to return to the previous screen.
Click Edit Trigger to edit the WS endpoint trigger.
Click Refresh Page to update the information on the screen.
JMS Destinations and Message Selectors
Use this section to view information about the destinations and messages selectors that make up the subscription for the trigger.
Destination Name
Destination from which the WS endpoint trigger receives messages.
Destination Type
Indicates whether the destination is a queue or a topic.
JMS Message Selector
The expression used to receive a subset of messages from the destination. The message selector specifies the criteria for the messages in which the WS Endpoint trigger is interested.
Durable Subscriber Name
For a topic that establishes a durable subscription, the name of the durable subscriber.
Ignore Local Published
Indicates whether the WS endpoint trigger ignores messages sent by the JMS connection alias used by the trigger.
State
This section displays information about the current state of the trigger.
Enabled
Indicates whether or not the WS endpoint trigger is enabled.
Current Threads
Number of threads currently used by the WS endpoint trigger. The value “Not Connected” indicates the WS endpoint trigger is not connected to the JMS provider.
Settings
This section displays the JMS connection alias and type of JMS trigger.
JMS Connection Alias
Name of the JMS connection alias used to retrieve messages from the JMS provider.
JMS Trigger Type
Indicates that the JMS trigger is a WS endpoint trigger, which is a SOAP-JMS trigger.
Properties
This section displays message processing properties for the JMS trigger.
Processing Mode
Specifies how the Integration Server processes the messages for the JMS trigger. Serial specifies that Integration Server should process messages one after the other, in the same order that they were received. Concurrent specifies that Integration Server should process as many messages as it can at one time.
Maximum Execution Threads
The number of threads currently being used to process message for this JMS trigger followed by, in parentheses) the maximum number of sever threads that can be used to process messages for this trigger at one time.
Connection Count
Maximum number of connections that the WS endpoint trigger makes to the JMS provider.
