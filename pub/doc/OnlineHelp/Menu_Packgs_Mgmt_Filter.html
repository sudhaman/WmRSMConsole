Filtering Packages Screen
Related Topics
Screen Heading: Packages > Management > Filter Packages
Use the package filter to limit the packages to be displayed, making the Packages List shorter and more manageable. You can include or exclude packages that match specified criteria, and narrow down a filtered list by filtering on the results again.
To enable package filtering, click Filter Packages on the Packages Management Screen.
n
Click Show All Packages to disable filtering and restore the default list of all packages on the server. This also updates the Packages List to reflect any changes that occurred to the Integration Server (such as new packages, etc.) while Filter Packages was enabled.
Filter criteria
Specifies the string to submit to the filter. By default, packages with names that match the string are included in the results. Filter criteria can be literals or a combination of literal and wild-card characters. The "*" (asterisk) and "?" (question mark) are the only supported wild-card characters. Note: Leaving the filter criteria blank includes all packages.
 
 
Important! The package names in the Filter criteria field are case‑sensitive. If you enter “wma*”, the filter will ignore any packages beginning with “WmA”.
Include Enabled
Includes only packages that are enabled in the results (that is, those that have a Yes in the Enabled column of the Packages List).
Include Disabled
Includes only packages that are disabled in the results, (that is, those with a No in the Enabled column of the Packages List).
Include Both
Includes all packages in the results, whether they are enabled or disabled.
Filter on result
Limits the search to just the currently displayed list of packages (that is, the results of the previous filtering operation), rather than the original list of all packages on the server.
Exclude from result
Displays the packages that do not match the Filter criteria, rather than the packages that do match them.
Submit
Processes the filter criteria and updates the page.
