VSS (Microsoft Visual SourceSafe) Configuration Screen
Screen Heading: VCS > Configuration > Edit VCS Configuration > VSS Configuration
Use the VSS Configuration screen to configure the connection from the Integration Server to a VSS (Microsoft Visual SourceSafe) server.
To open the VSS Configuration screen, click VCS under the Solutions menu on the Navigation panel and select Edit Configuration on the Configuration screen. Verify that Microsoft Visual SourceSafe is specified in the Select Version Control System drop-down menu and click Advanced Settings.
Click Return to Edit VCS Configuration to return to the previous screen.
Click the Save Changes button to save any changes you have made.

Command Timeout (msec)
Period of time, in milliseconds, that a command has in which to execute. If the command does not execute within that period, it times out.
If you want to use the default of 60000 (one minute), leave this field blank. To change the default, specify a positive integer. If you want to have no timeout, enter -1.
Note:  
The timeout is applied to each command. When Visual SourceSafe operations join several commands, the timeout is the sum of the timeouts for all the commands.
Working Folder
Fully qualified path to the Visual SourceSafe working folder in the Integration Server file system in which to store your working files. Software AG recommends that you specify the Integration Server_directory \instances\instance_name\packages directory, or an individual package in that directory.
VSS Project Name
Name of the project in Visual SourceSafe that contains all of the files and directories you work on. Typically, this path begins with a dollar sign ($), to indicate the Visual SourceSafe root project (for example: $/Project/Team/Module).
Allow Multiple Checkouts
If Visual SourceSafe is configured for exclusive check out, clear this check box. If Visual SourceSafe is configured for non-exclusive check out, select this check box.
Important!  
Software AG recommends that you configure your VCS for exclusive locking to prevent users from checking out files from the VCS client after they have been checked out from Designer. The VCS Integration feature does not support "diff" or merge operations. If you use non-exclusive checkout and multiple users check out the same element, only the first user to check in will be successful. The other users will receive error messages.
