Edit OAuth Global Settings Screen
Screen Heading: Security > OAuth > Edit OAuth Global Settings
Use the Edit OAuth Global Settings screen to specify the settings for the application server.
To open the Edit OAuth Global Settings screen, go to the OAuth Screen and click Edit OAuth Global Settings.
Click Return to OAuth to return to the OAuth Screen.
Authorization Server Settings
Require HTTPS
Indicates whether the authorization server should require an HTTPS connection to authorize requests. If enabled (the default), Integration Server requires that the authorization server uses HTTPS to invoke the pub.oauth:authorize, pub.oauth:getAccessToken, and pub.oauth:refreshAccessToken services.
Important!  
If Require HTTPS is disabled, Integration Server will issue an exception.
Authorization code expiration interval
Specifies the length of time (in seconds) that the authorization code issued by the authorization server is valid. Valid values are between 1 and 2147483647. The default value is 600.
Access token expiration interval
Specifies the length of time that access tokens issued by the authorization server are valid. Select one of the following:
Option
Description
Never Expires
Indicates that the access token never expires.
Expires in
Specifies the length of time (in seconds) that the access token is valid. The maximum value is 2147483647. The default is 3600.
Resource Server Settings
Authorization server
Indicates the Integration Server to use as the authorization server. If you have not already defined an alias for the authorization server, click the link to go to the Remote Servers Screen.
The list displays the configured remote server aliases that are available for use. For more information about creating a remote server alias, see webMethods Integration Server Administrator’s Guide .
Save Changes
Updates Integration Server with the new or changed OAuth settings. Your changes take effect immediately.
