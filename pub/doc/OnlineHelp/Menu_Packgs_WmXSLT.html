WmXSLT Package Transformer Factory Settings Screen
Screen Heading: WmXSLT > Transformer Factory Settings > Edit
Use the WmXSLT screen to configure the transformer factory settings to point to the appropriate XSLT processors.
When executed, an XSLT service calls an external XSLT engine to convert the XML data. By default, the Integration Server includes the Xerces parser and the Xalan XSLT stylesheet processor from the Apache Software Foundation. Also included is the open source Saxon-B XSLT processor from Saxonica, which supports both XSLT 1.0/2.0 and XPath 1.0/2.0.
To use a more current version of the Xalan, Xerces, Saxon, or other 3rd-party XSLT processor, you must place all the related jar files in the Integration Server_directory \instances\instance_name\packages\WmXSLT\code\jars directory and configure the transformer factory settings to point to the appropriate XSLT transformer class.
For more information about transformer factory settings and XSLT services, see the webMethods Service Development Help .
To open the WmXSLT screen, select Management on the Packages menu. The Packages Management Screen will be displayed. Click the Home icon next to the WmXSLT package name.
Click Edit Transformer Factory Settings to change the fields below.
Transformer Factory Settings
Transformer Factory
Specify a transformer factory provided with Integration Server, or select Other and type in the name of the factory you want to use.
Transformer Factory Attributes
Specify any factory attributes that you want to use. The attributes must take the form attribute=value. You can only specify attributes that accept strings or Boolean values. For more information about possible attributes, refer to the documentation from your XSLT processor vendor.
