Edit CSRF Settings Screen
Screen Heading: Security > CSRF Guard > Edit CSRF Guard Settings
Use the Edit CSRF Guard Settings screen to enable or disable CSRF guard and to configure the CSRF guard settings in Integration Server.
To open the Edit CSRF Guard Settings screen, click CSRF Guard under the Security menu on the Navigation panel. Click Edit CSRF Guard Settings.
Click Return to CSRF Guard Settings to return to the CSRF Guard Settings page.
Enabled
Specifies whether CSRF guard is enabled in Integration Server. Select the Enabled check box to enable CSRF guard in Integration Server. Clear the Enabled check box if you do not want to use CSRF guard in Integration Server. CSRF guard is disabled by default.
Excluded User Agents
Enter the user agents for which CSRF guard is not to be applied. If CSRF guard is enabled, Integration Server requires that HTTP requests coming from user agents that are not specified in this field must contain CSRF secure tokens.
You can specify these values as regular expressions. The asterisk (*) is the only wildcard character allowed in the regular expression. To separate the entries, enter one user agent per line. Press ENTER to separate the lines.
Landing Pages
Enter the list of landing pages for the packages in your Integration Server. A landing page is the home page for a package. Integration Server does not check for CSRF secure tokens in the landing pages, but inserts a token for that page. Integration Server guards all further requests from these landing pages with CSRF secure tokens.
You cannot specify landing pages as regular expressions. To separate the entries, enter one user agent per line. Press ENTER to separate the lines.
Note:  
Do not specify a landing page in the Unprotected URLs text area. If a landing page URL is specified in both Landing Pages and Unprotected URLs text areas, the landing page option will take precedence and Integration Server will not check for CSRF secure token in that page.
Unprotected URLs
Enter the URLs for which Integration Server does not have to check for CSRF secure tokens. Integration Server requires that the requests coming from all URLs that are not specified in this field must contain CSRF secure tokens.
If you specify a DSP file as an unprotected URL, Integration Server does not insert CSRF secure token for that file. If you attempt to access any other protected page from this DSP page, Integration Server issues an error or redirects you to the home page of Integration Server Administrator depending on your Denial Action configuration.
You can specify unprotected URLs as regular expressions. The asterisk (*) is the only wildcard character allowed in the regular expression. To separate the entries, enter one URL per line. Press ENTER to separate the lines.
Denial Action
Select the action you want Integration Server to perform when it detects that a request does not contain a CSRF secure token or contains an invalid CSRF secure token.
Select Error if you Integration Server to throw an access denied error and terminate the request. This is the default.
If you select Error, Integration Server issues the following error when it detects that a request does not contain a CSRF secure token or contains an invalid CSRF secure token:
Access Denied. Invalid CSRF secure token.
This error message indicates that Integration Server has detected a CSRF attack. However, Integration Server might throw this error message due to the following reasons as well:
Integration Server session has expired.
Integration Server Administrator is not refreshed after enabling CSRF guard.
Another user who is connected to the same Integration Server restarted Integration Server.
In these cases, refresh Integration Server Administrator to continue.
Select Redirect to redirect the user to a confirmation page or the home page of Integration Server Administrator.
If the CSRF threat is detected when accessing a DSP page, user will be redirected to the home page of Integration Server Administrator.
If the threat detected is in an invoke or rest directive, Integration Server redirects the user to a confirmation page. The confirmation page displays a warning that Integration Server has detected a CSRF attack. Click Continue to execute the service.
Save Changes
Click to save the CSRF guard settings.
