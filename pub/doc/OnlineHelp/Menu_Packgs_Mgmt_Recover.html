Recover Packages Screen
Screen Heading: Packages > Management > Recover Packages
Use the Recover Packages screen to recover a previously deleted package from the recovery directory ( Integration Server_directory \instances\instance_name\replicate\salvage). You can recover a deleted package only if you used the "safe delete" option to delete it. You can also use this screen to revert to the previous version of a package if you install a new version of the package that you are dissatisfied with.
To open the Recover Packages screen, open the Packages Management Screen and click Recover Packages.
Click Return to Package Management to return to the main package management screen.
Package name
Lists the packages that are available for recovery.
Option-Activate upon recovery
Specifies whether you want the server to activate the package immediately after the package is recovered. If you want the package to be available to clients immediately after it is recovered, enable this option. If you want to recover the package, but activate it at a later time, disable this option. (If you do not activate the package upon installation, use the Activate Inactive Packages Screen to activate it manually when you are ready to make it available to users.)
Recover
Recovers the package specified in Package Name.
