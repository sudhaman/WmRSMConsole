View Keystore Alias Screen
Screen Heading: Security > Keystore > View Keystore Alias
Use the View Keystore Alias screen to view the properties of a specific keystore used by Integration Server. A keystore contains private keys and certificates for an Integration Server, partner application, or Integration Server component. A keystore alias is a user-specified identifier that can be easily referenced from within Integration Server.
To open the Keystore screen, click Keystore under the Security menu on the Navigation panel. Select the name of a keystore alias to view its properties.
Click Return to Keystore to return to the list of keystore aliases that are currently defined on the server.
Click Edit Keystore Alias to edit the properties of the keystore alias currently displayed.
Keystore Properties
Alias
Displays the user-specified identifier for the keystore. Clicking on the keystore alias displays its properties.
Description (optional)
Displays a user-specified description of the alias.
Type
Displays the file format of the keystore. The default is JKS. You can also use PKCS12 format. Other keystore types can be made available by loading additional security providers, or by setting the watt.security.keyStoreTypes server configuration property.
Provider
Displays the provider that is used for the keystore type. The default is the provider shipped with the JVM (Oracle, IBM, or others). If your Hardware Security Module (HSM) device is not supported by the default provider, you can configure a different provider to support keystore file types.
Note:  
Integration Server supports both PKCS12 and JKS file formats for keystore files.
Location
Displays the location of the keystore file on the server. This can be the full pathname, or a relative path in relation to the Integration Server.
HSM Based Keystore
Displays whether or not the keystore is backed by an HSM-based keystore (a smart card device can be used as well). When the keystore is backed by such a device, the private key does not physically leave the HSM device and certain cryptographic operations must be performed on that device.
Configured Key Aliases
Displays a list of all keystore alias names defined on Integration Server.
