Package Description Screen
Screen Heading: Packages > Management > packageName
Use the Package Description screen to view detailed information about a package.
To open the Package Description screen, go to the Packages Management Screen and click the package name.
Click Return to Package Management to return to the main package management screen.
Click Browse services in packageName to display the list of services in this package.
Package Information
This section displays details about the selected package.
Package Name
Displays the name of the package.
Version
Displays the version number of the package.
Build
Displays the build number of this package, if one was assigned by the publisher of the package.
Minimum Version of JVM
Indicates the minimum version of the Java Virtual Machine (JVM) required to run this package, if specified by the publisher of the package.
Package List ACL
Name of the Access Control List assigned to the package. Users associated with this ACL can see the package listed on Integration Server Administrator or Designer. To see the folders and elements contained in the package, a user must have List access to the folders and elements themselves.
Patches Included
Displays the list of patches that have been applied to this release of the package, if this information was provided by the publisher of the package.
Description
Displays a description of the package, if this information was provided by the publisher of the package.
Publisher
Displays the name of the publisher.
Created on
Displays the date that this release of the package was generated.
Elements Loaded
Displays the number of elements that loaded successfully. To view the list of these elements, click the Browse Services in packageName link at the top of the screen.
Elements Not Loaded
Displays the number of elements that did not load successfully. To view the list of these elements, look at the Load Errors list at the bottom of this screen.
Startup Services
Displays the names of startup services assigned to the package. A startup service is one that the server executes automatically when it loads a package. To assign a startup service to a package, you use Software AG Designer. For procedures, see webMethods Service Development Help .
Shutdown Services
Displays the names of shutdown services assigned to the package. A shutdown service is one that the server executes automatically when it unloads a package from memory. To assign a shutdown service to a package, you use Software AG Designer. For procedures, see webMethods Service Development Help .
Replication Services
Displays the name of replication services assigned to the package. A replication service is one that the server executes automatically when it prepares to release or archive a package. Replication services provide a way for a package to persist state or configuration information so that this is available when the package is activated on the remote server. To assign a replication service to a package, you use Software AG Designer. For procedures, see webMethods Service Development Help .
Packages on which this package depends
Displays the names of packages that must be loaded before this package is loaded. To specify package dependencies, you use Software AG Designer. For procedures, see webMethods Service Development Help .
Packages that depend on this package
Displays a list of packages that depend on this package. If you disable the package, these packages will be affected. To specify package dependencies, you use Software AG Designer. For procedures, see webMethods Service Development Help .
Subscribers
Displays the list of other webMethods Integration Servers that subscribe to this package.
Load Errors
Displays a list of services that generated errors and could not be loaded onto the server when the package was installed. When some services do not load, the load status for the package becomes Partial.
Load Warnings
Displays a list of services that generated warnings when the package was installed. The server was able to load the packages, despite the warnings. When package elements are loaded with warnings, the load status for the package becomes Warnings.
Patch History
Displays any partial packages that have been applied to this package.
