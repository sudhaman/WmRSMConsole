Edit Alert Options Screen
Screen Heading: Security > Mobile Gateway > Edit Default Alert Options
Use the Edit Alert Options screen to configure the alert options.
To open the Edit Alert Options screen, click Mobile Gateway under the Security menu on the Navigation panel. Click Edit Default Alert Options.
You can also open this screen by clicking the Default or Custom link in the Alert Options column in the Mobile Gateway screen.
Click Return to Mobile Gateway to return to the Mobile Gateway page.
Alert Type
Select one of the following options:
None if you do not want Integration Server to send any alerts. This is the default.
E-mail if you want Integration Server to send e-mail alerts.
Flow Service if you want Integration Server to invoke a flow service to alert you of a rule violation.
Send Alert
Specify whether Integration Server issues the alerts every time a request violates a Mobile Gateway rule or at any specific time interval (in minutes). If no rules are violated within the specified interval, no e-mail alert will be issued.
E-mail IDs
Enter the e-mail IDs to which you want Integration Server to send the alerts if you selected e-mail as the alert type. You can enter multiple e-mail IDs. Separate e-mail IDs with semicolons (;).
folder.subfolder:service
Enter the fully qualified service name of the flow service you want Integration Server to execute if you selected flow service as the alert type.
Note: 
Use the pub.security.mobilegateway:alertSpec specification as the signature for the flow service. For more details about the specification, see webMethods Integration Server Built-In Services Reference.
Run As User
Specifies the user name under which the service will run (e.g., Administrator). If the service is protected by an ACL, be sure to specify a user name that is authorized to invoke the service.
Save Changes
Saves the alert configuration.
