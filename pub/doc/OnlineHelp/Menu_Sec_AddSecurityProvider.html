Add a Security Provider
Screen Heading: Security > Keystore > Add Security Provider
Use the Add Security Provider screen to configure a new Security Provider that will provide support for the keystore type you want to use.
If you want to add an HTTPS or FTPS port with a listener that will use a private key and certificate chain residing in a keystore, and the keystore is managed by a non-standard Security Provider, you may need to add that Security Provider to the Integration Server Administrator.
When specifying keystore information in the HTTPS or FTPS port information screen, a non-standard Security Provider may not appear in the KeyStore Type parameter drop-down list. If the Security Provider that you want to use does not appear in the list, you can add it by clicking the Add New Security Provider link and entering the fully qualified name of the security provider class.
To open the Add Security Provider screen, go to the Keystore Screen and click Add Security Provider.
Click Return to Keystore to return to the list of ports that are currently defined on the server.Security Provider Class Enter the Java class name of the security provider used for additional keystore and truststore file types. If the keystore type supported by the HSM (Hardware Security Module) is different than that supported by the Integration Server default (JKS for truststores and keystores), then use this field to specify the supporting security provider. For nCipher, for example, the fully qualified name is com.ncipher.provider.km.nCipherKM.

Add Provider
The server adds the security provider to the KeyStore Type drop-down list and adds the security provider class to the java.security file for your JDK.
