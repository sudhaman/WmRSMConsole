Edit Reverse HTTP Gateway Server Configuration Screen
Screen Heading: Security > Ports > Edit Reverse HTTP Gateway Configuration
Use this screen to implement the HTTP Gateway behavior, in which the Reverse Gateway Server forwards all inbound requests to an Internal Server through the Reverse Gateway Server's gateway registration port. Refer to Administering webMethods Integration Server for more information.
To open the Edit Reverse HTTP Gateway Configuration screen, go to the Add Port Screen and select Reverse HTTP Gateway Server. Click Submit.
Click Return to Ports to return to the list of ports that are currently defined on the server.
Gateway External Port
Enable
Controls whether or not the port is enabled. When the port is enabled, it can receive requests.
Note: 
A disabled port is not re-enabled by restarting the server. A disabled port does not accept requests until you explicitly enable it.
Protocol
Select HTTP or HTTPS. If you select HTTPS, additional security and credential fields will be displayed at the bottom of the screen.
Port
The number you want to use for the gateway external port. Select a number that is not already in use. This is the port that will be allowed through your outer firewall.
Package Name
The package associated with this port. The default package is WmRoot. Typically you won't need to work with packages on a Reverse HTTP Gateway Server; therefore, you can leave this field with the default setting.
Bind Address (optional)
The IP address to which you want to bind this port. Specify a bind address if your machine has multiple IP addresses and you want the port to use a specific address. If you do not specify a bind address, the server picks one for you.
Backlog
The number of requests that can remain in the queue for a suspended port before Integration Server begins rejecting requests. The default is 200. The maximum value is 65535.
Keep Alive Timeout
How long to wait before closing an idle connection to a client. The default is 20000 ms.
Threadpool
Whether or not this server will use a private threadpool for this port.
If you select Disable, the server uses the common server threadpool for this port.
If you select Enable, the server creates a private threadpool for this port so that it does not need to compete with other server functions for threads.
Option
Description
Threadpool Min
Minimum number of threads the server maintains in this threadpool. When the Reverse Gateway Server starts, the threadpool initially contains this minimum number of threads. The server adds threads to the pool as needed until it reaches the maximum allowed. The default is 1.
Threadpool Max
Maximum number of threads the server maintains in this threadpool. If this maximum number is reached, the server waits until services complete and return threads to the pool before running more services. The default is 5.
Threadpool Priority
Priority with which the JVM treats threads from this threadpool. The larger the number, the higher the priority. The default is 5.
Important! 
Be very careful when setting the threadpool priority; it can affect server performance and throughput.
When you view the port’s details, the server reports the total number of private threadpool threads currently in use for the port.

Security Configuration
Client Authentication
The type of client authentication to perform for requests coming through the gateway external port (in other words, requests coming from the external client). For more information about authenticating clients or about using the Reverse HTTP Gateway Server to perform client authentication, see Administering webMethods Integration Server.
Username/Password. The Reverse HTTP Gateway Server will not request client certificates. Instead it looks for user and password information in the request header.
Request Client Certificates. The Reverse HTTP Gateway Integration Server will request client certificates for requests that come through this port (the gateway external port). If the client does not present a certificate, the request proceeds using the user and password information contained in the request header.
Require Client Certificates. The Reverse HTTP Gateway Server requires client certificates for all requests that come through this port (the gateway external port). If the client does not supply a certificate, the request fails.
Listener Specific Credentials (Optional)
Keystore Alias
Optional. Specifies a text identifier for the keystore, which contains private keys and certificates for an Integration Server, partner application, or Integration Server component.
Key Alias
Optional. Specifies a text identifier for the private key associated with the keystore alias.
Truststore Alias
Optional. Specifies a text identifier for the truststore, which contains the trusted root of a certificate or signing authority (CA).
Gateway Registration Port
Enable
Controls whether or not the port is enabled. When the port is enabled, it can receive requests.
Note: 
A disabled port is not re-enabled by restarting the server. A disabled port does not accept requests until you explicitly enable it.
Protocol
Select HTTP or HTTPS. If you select HTTPS, additional security and credential fields will be displayed at the bottom of the screen.
Port
The number you want to use for the gateway registration port.
Package Name
The package associated with this port.The default package is WmRoot. Typically you won't need to work with packages on a Reverse Gateway Integration Server; therefore you can leave this field with the default setting.
Bind Address (optional)
The IP address to which you want to bind this port. Specify a bind address if your machine has multiple IP addresses and you want the port to use a specific address. If you do not specify a bind address, the server picks one for you.
Security Configuration
Client Authentication
The type of client authentication to perform when the Internal Server establishes a persistent connection to the Reverse Gateway Integration Server. This setting controls whether the Reverse HTTP Gateway Server will ask the Internal Server to present a certificate. See Administering webMethods Integration Server for more information about how clients are authenticated.
Username/Password. The Reverse HTTP Gateway Server will not request client certificates. Instead it uses user and password information.
Request Client Certificates. The Reverse HTTP Gateway Server will request a client certificate from the Internal Server. If the Internal Server does not present a certificate, the request proceeds using the user and password information.
Require Client Certificates. The Reverse HTTP Gateway Server requires a client certificate from the Internal Server. If the Internal Server does not supply a client certificate, the request fails. In addition, if the certificate is not mapped to a user with Administrator privileges on the Reverse HTTP Gateway Server, the request fails.
Listener Specific Credentials (Optional)
Note: 
If you selected HTTPS in the Protocol field under Gateway Registration Port, the following fields are enabled:
Keystore Alias
Optional. Specifies a text identifier for the keystore, which contains private keys and certificates for an Integration Server, partner application, or Integration Server component.
Key Alias
Specifies a text identifier for the private key associated with the keystore alias.
Truststore Alias
Specifies a text identifier for the truststore, which contains the trusted root of a certificate or signing authority (CA).
Save Changes
Updates the server with the new or changed port settings. Your changes take effect immediately.
