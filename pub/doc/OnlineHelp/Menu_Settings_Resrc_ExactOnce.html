Exactly Once Statistics Screen
Screen Heading: Settings > Resources > Exactly Once Statistics
Use the Exactly Once Statistics screen to manually remove expired or duplicate information from the document history database. The document history database maintains a record of documents processed by triggers as part of duplicate detection or exactly-once processing.
Note:  
Using the Remove Expired Document History Entries link to clear expired entries does not affect the next scheduled execution time for the wm.server.dispatcher:deleteExpiredUUID service.
To open the Exactly Once Statistics screen, click Resources under the Settings menu on the Navigation panel. On the Resources Screen, click Exactly Once Statistics.
Click Return to Resource Settings to return to the list of current resource settings.
Click Remove Expired Document History Entries to remove all expired entries from the document history database. These entries are for documents processed by triggers that specify that document history should be used as part of duplicate detection.
Click Clear All Duplicate or In Doubt Document Statistics to remove the currently displayed history of the In Doubt or Duplicate documents. These documents were received by triggers for which exactly-once processing is configured.
Duplicate or In Doubt Document Statistics
A list of documents whose status is either Duplicate or In Doubt.
