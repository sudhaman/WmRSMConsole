Broker Settings Screen
Screen Heading: Settings > Messaging > Broker Settings
Use the Broker Settings screen to view Broker connection information.
To open the Broker screen, click Messaging under the Settings menu on the Navigation panel. Under BrokerConfiguration on the Messaging Screen, click Broker.
Click Return to Messaging to return to the previous screen.
Click Edit Broker Settings to configure an Integration Server to connect to a Broker or to edit the existing Broker connection.
Broker Connection
Broker Host
Name (DNSname:port or ipaddress:port) of the machine on which the Broker server resides.
Broker Name
Name of the Broker as defined on the Broker Server. The default is Broker #1.
Client Group
Client group to which this Integration Server belongs. A client group defines a single set of properties and access permissions assigned to one or more clients (here, Integration Servers) on a single Broker. If the specified client group does not exist, the Integration Server creates it on the named Broker upon establishing its initial connection.
Note: 
Brokers do not share can-publish and can-subscribe permissions across client groups. If you switch an Integration Server from one client group to another, you must restart the Integration Server and synchronize all publishable document types with the Broker. Next, you must shut down the server and use My webMethods to delete all of the Broker clients created for the server with the changed client group. Restart the server with the changed client group.
Client Prefix
A string that identifies the Integration Server to the Broker. By default, the server generates an obscure value for the prefix. For ease of use, you may want to replace it with a friendly name.
The Broker Manager displays this prefix for each client it creates for the server. (The Broker creates multiple clients for each server that connects to it.)
Share Client Prefix
Whether you want the Integration Server to share a client prefix. Yes indicates that the client prefix may be shared among multiple Integration Servers. Integration Servers that share a client prefix can connect to a single Broker and operate in a load-balanced fashion without being in a cluster.
If the Integration Server belongs to a cluster, this option is automatically enabled.
If the Integration Server does not belong to a cluster, you can enable or disable this option.
Note: 
When this option is enabled on non-clustered servers, you cannot modify, delete, or reload subscriptions for publishable documents using Integration Server Administrator. You must do so from the trigger client queue on the Broker using My webMethods Server.
Client Authentication
Whether you want Integration Server clients to connect to the Broker Server using basic authentication (user name and password), Secure Sockets Layer (SSL), or an anonymous connection. If you select the Username/Password option, you are required to enter the user name and password the client will use to connect to the Broker Server. If you select the SSL option, you are required to fill in the Keystore and Truststore fields. If you choose None, clients can connect to the Broker Server anonymously.
Encryption
Whether you want Integration Server to encrypt the connection between the Integration Server and Broker. When you set Client Authentication to SSL, Encryption must be set to Yes. If you select Yes for the Encryption option, you are required to fill in the Truststore fields.
Connected
Whether the Integration Server is connected to Broker Servers currently.
