System Threads Screen
Related Topics
Screen Heading: Server > Statistics > System Threads
Use the System Threads screen to view the status of the server's system threads.
To open the System Threads screen, first go to the Server Statistics Screen. In the Usage section, find the System Threads row and click the number link in the Current column.
n
Click Return to Server Statistics to return to the main statistics screen.
Priority
Displays the order in which the threads are run. Threads with a higher priority run before those with lower priority, e.g., threads with a priority of 5 run before threads with a priority of 4. The threads with the same priority level can be scheduled to run one at a time or to run concurrently, depending on what the Java Virtual Machine can handle.
Name
Displays the name of the thread.
Alive
Indicates whether or not the thread has been started and has not died. A green checkmark indicates that the thread has been started.
Daemon
Indicates whether or not the thread is a daemon (services other threads). A green checkmark indicates that the thread is a daemon.
Interrupted
Indicates whether or not the thread has received an interrupt signal, which may indicate that an error has occurred. If your server is not behaving as expected, the status of this field can indicate whether an internal process has malfunctioned.
