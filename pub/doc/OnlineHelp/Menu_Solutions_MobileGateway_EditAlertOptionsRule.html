Edit Alert Options (Rule)
Screen Heading: Mobile Gateway > Rules > Edit Alert Options > rulename
Use the Edit Alert Options screen to edit the alert options for this rule.
To open the Edit Alert Options screen for a specific rule, click Mobile Gateway under the Solutions menu, click Rules, then click the underlined text in the Alert Options field for the rule you want to update.
Click Return to Rules to return to the Rules screen.
Alert Settings
Alert options Integration Server is to use when it sends an alert about a violation of this rule. If Default is selected, Integration Server uses the alert settings defined on the Default Alert Options screen. If Custom is selected, Integration Server uses custom settings that were defined specifically for this rule.
Alert Type
Type of alert Integration Server issues when a request violates this rule. Can be None, E-mail, or Flow Service.
Send Alert
Frequency with which Integration Server sends an alert when a rule is violated.
Select...
To...
On Rule Violation
Send an alert each time a request violates the rule.
Every number minutes
Limit the number of alerts sent within a specified time interval. When you select this option, Integration Server sends an alert the first time a request violates a rule. If the violation occurs again within the specified time interval, Integration Server waits until the end of the time interval before sending another alert. For example, suppose you specify the time interval as 1 minute. At 10:00 a request violates the rule and Integration Server sends an alert. Ten seconds later another request violates the rule. Ten seconds after that yet another request violates the rule. Integration Server waits until 10:01 to send a summary of alerts that were generated in the time interval.
E-mail IDs
One or more E-mail IDs to which Integration Server sends an alert when a request violates this rule. Specify e-mail IDs only if the alert type is E-mail. To use this option, the Reverse HTTP Gateway Server on which Mobile Gateway runs must be configured to send e-mails. Check the settings on the Settings > Resources screen under Email Notification. If your Integration Server is not configured to send e-mails, see Administering webMethods Integration Server for instructions on configuring Integration Server to have this ability.
folder.subfolder:service
Name of the flow service to run when a request violates a this rule. Specify a value in this field only if the alert type is Flow Service. Use the pub.security.mobilegateway:alertSpec specification as the signature for the flow service. For more information about the specification, see webMethods Integration Server Built-In Services Reference.
Run As User
Integration Server user ID to run the flow service configured to execute when a request violates this rule. Specify a Run As User only if the alert type is Flow Service.
