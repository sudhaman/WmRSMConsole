webMethods Messaging Trigger Detail Screen
Screen Heading: Settings > Messaging > webMethods Messaging Trigger Management > triggerName
To open the webMethods Messaging Trigger Management screen, click Messaging under the Settings menu on the Navigation panel.
On the Messaging Screen, click webMethods Messaging Trigger Management. On the webMethods Messaging Trigger Management Screen, in the Trigger Name column, click the trigger name whose details you want to view.
Click Return to webMethods Messaging Trigger Management to return to the previous screen.
Click Edit Properties to change the configuration information for the webMethods messaging trigger.
Click Refresh Page to update the information on the screen.
Conditions
Name
Name of the condition in the webMethods messaging trigger.
Service
Name of the service thatIntegration Server invokes when it receives documents that satisfy the trigger condition.
Join
Specifies the join type for the trigger condition. This field displays N/A if the trigger condition specifies only one document type.
Document
Name of the publishable document type to which the trigger subscribes.
Filter
Specifies the filter that Integration Server applies to the subscribed document. If the document does not meet the filter criteria, Integration Server discards the document. Click view to display the filter in a separate window.
State
Retrieval State Active
Indicates whether document retrieval is active or suspended for this trigger. Yes indicates that document retrieval is active. No indicates that document retrieval is suspended. An asterisk (*) appears next to the status if the document retrieval state is temporary.
Processing State Active
Indicates whether document processing is active or suspended for this trigger. Yes indicates that document processing is active. No indicates that document processing is suspended. An asterisk (*) appears next to the status if the document processing state is temporary.
Current Threads (Processing)
Specifies the current number of server threads processing documents for this trigger.
Volatile Queue
Specifies the current number of documents in the volatile queue for the trigger. The volatile queue contains the volatile documents received by the trigger.
Persisted Queue
Specifies the current number of documents in the persisted queue for the trigger. The persisted queue contains the guaranteed documents received by the trigger.
Properties
Enabled
Specifies whether the trigger is enabled or disabled. True indicates that the trigger is enabled. An enabled trigger contains one or more valid conditions. False indicates that the trigger is disabled. A disabled trigger can contain valid or invalid conditions. A disabled trigger does not have any subscriptions registered on Integration Server or the messaging provider. Broker.
Join Expires After
Indicates the number of milliseconds after which the join expires.
Trigger Queue Capacity
Specifies the current maximum capacity for the trigger queue. If the current queue capacity is different from the configured queue capacity (that is, the Queue Capacity Throttle is set to less than 100%), the configured queue capacity is displayed in parenthesis.
Trigger Queue Refill Level
Specifies the current refill level for the trigger queue. If the current refill level is different from the configured refill level (that is, the Queue Capacity Throttle is set to less than 100%), the configured refill level is displayed in parenthesis. Refill level does not apply to webMethods messaging triggers that receive documents from Universal Messaging.
Acknowledgment Queue Size
Specifies the maximum number of pending document acknowledgements for this trigger. A server thread places acknowledgements in the acknowledgement queue after it finishes executing the trigger service. Acknowledgements collect in the queue until the server returns them as a group to the sending resource. If the acknowledgement queue fills to capacity, the server blocks any server threads that attempt to add an acknowledgement to the queue. The blocked threads resume execution only after Integration Server empties the queue by returning the acknowledgements.
Maximum Retry Attempts
Specifies the maximum number of times Integration Servercan re-execute the trigger service if the trigger service fails because of an ISRuntimeException.
Retry Interval
Specifies the number of milliseconds that Integration Server waits between attempts to re-execute the trigger service.
Retry Failure Behavior
Specifies how Integration Server handles retry failure for this trigger. Retry failure occurs when Integration Server makes the maximum number of retry attempts and the trigger service still fails because of an ISRuntimeException.
Throw service exception Integration Server throws a service exception when the last allowed retry attempt ends because of an ISRuntimeException.
Suspend and retry later Integration Server suspends the trigger when the last allowed retry attempt ends because of an ISRuntimeException. Integration Server retries the trigger service at a later time.
Resource Monitoring Service
Specifies the service that Integration Server executes to determine the availability of resources associated with the trigger service. Multiple triggers can use the same resource monitoring service.
Processing Mode
Specifies how Integration Server processes the documents in the trigger queue. Serial specifies that tIntegration Server processes documents in the trigger queue one after the other, in the same order that they were received. Concurrent specifies that Integration Server processes as many documents in the trigger queue as it can at one time.
Maximum Execution Threads
Specifies the maximum number of server threads that can process documents for this trigger concurrently. If the current maximum threads value is different from the configured maximum threads value (that is, the Execution Threads Throttle is set to less than 100%), the configured maximum threads value is displayed in parenthesis. This value applies to concurrent triggers only.
Suspend on Error
Specifies whether Integration Server suspends document processing and document retrieval automatically when a trigger service ends with an exception. True indicates that the trigger suspends on error. Only serial triggers can be configured to suspend on error.
Exactly Once Detect Duplicates
Indicates whether Integration Server uses duplicate detection to ensure exactly-once processing for documents received by this trigger. True specifies that exactly-once processing is provided for documents received by this trigger and instructs the server to check a document's redelivery count to determine whether the trigger received the document previously. False specifies that exactly-once processing is not provided for documents received by this trigger.
Exactly Once Use History
Indicates whether a document history will be maintained and used to determine whether a document is a duplicate. True indicates that Integration Server uses document history as part of duplicate detection. False indicates that Integration Server does not use document history.
Exactly Once History Time to Live
Specifies the length of time the document history database maintains an entry for a document processed by the trigger.
Priority Enabled
Specifies whether priority messaging for the trigger is enabled or disabled. True indicates that priority messaging for the trigger is enabled. False indicates that priority messaging for the trigger is disabled. Priority messaging is disabled by default.
