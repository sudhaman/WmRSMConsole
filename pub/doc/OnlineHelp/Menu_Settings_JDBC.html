JDBC Pools Screen
Screen Heading: Settings > JDBC Pools
Use the JDBC Pools screen to define a connection pool for each Integration Server function.
To open the JDBC Pools screen, click JDBC Pools under the Settings menu on the Navigation panel.
Click Create a New Pool Alias Definition to identify a database to the Integration Server.
Click Create a New Driver Alias Definition to identify a database driver to the Integration Server.
Click Edit to change the definition of a specific functional, pool, or driver alias.
Click the icon in the Test column to validate the connection to the database, using the connection information defined in the pool alias.
Click Restart to refresh the connection information for the database.
Functional Alias Definitions
A functional alias directs auditing data and other information (such as the Trading Networks Audit Log and Key Cross Referencing and Echo Suppression) to a particular JDBC database. Functional aliases are predefined to the server. You cannot add or delete functional aliases, but you can change the JDBC connection pool to which a functional alias is assigned. In this section of the JDBC Pools screen, you can also view the minimum and maximum number of JDBC pool connections for each functional alias, the current total number of connections created for the JDBC pool, and the available connections.
Pool Alias Definitions
The pool alias defines the database's location and connection parameters. The server uses this information to interact with the database resource.
Driver Alias Definitions
The driver alias identifies a database driver to the server. The server uses this database driver to connect to the defined database resource.
