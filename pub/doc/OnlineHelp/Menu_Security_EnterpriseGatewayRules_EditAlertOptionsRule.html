Edit Rule Alert Options
Screen Heading: Security > Enterprise Gateway Rules > Rules > rulename Alert Options > Edit
Use the Edit Alert Options screen to edit the alert options for this rule.
To open the Edit Alert Options screen for a specific rule, click Enterprise Gateway Rules under the Security menu, and then click the underlined text in the Alert Options field for the rule you want to update.
Click Return to Rules to return to the Enterprise Gateway Rules screen.
Alert Settings
Alert options Enterprise Gateway Server is to use when sending an alert about a violation of this rule. If Default is selected, the server uses the alert settings defined on the Default Alert Options screen. If Custom is selected, the server uses custom settings that were defined specifically for this rule.
Alert Type
Type of alert Enterprise Gateway Server issues when a request violates this rule. Can be None, Email, or Flow Service.
Send Alert
Frequency with which Enterprise Gateway Server sends an alert when a rule is violated.
Select...
To...
On rule violation
Send an alert each time a request violates the rule.
Every number minutes
Limit the number of alerts sent within a specified time interval. When you select this option, Enterprise Gateway Server sends an alert the first time a request violates a rule. If the violation occurs again within the specified time interval, the server waits until the end of the time interval before sending another alert. For example, suppose you specify the time interval as 1 minute. At 10:00, a request violates the rule and the server sends an alert. Ten seconds later, another request violates the rule. Ten seconds after that, yet another request violates the rule. The server waits until 10:01 to send a summary of alerts that were generated in the time interval.
Email Addresses
One or more email addresses to which Enterprise Gateway Server sends an alert when a request violates this rule. Specify email addresses only if the alert type is Email. To use this option, the Integration Server acting as the Enterprise Gateway Server must be configured to send emails. Check the settings on the Settings > Resources screen under Email Notification. If Enterprise Gateway Server is not configured to send emails, see webMethods Integration Server Administrator’s Guide for instructions on configuring Enterprise Gateway Server to have this ability.
folder.subfolder:service
Name of the flow service to run when a request violates a this rule. Specify a value in this field only if the alert type is Flow Service. Use the pub.security.enterpriseGateway:alertSpec specification as the signature of the flow service. For more information about the specification, see webMethods Integration Server Built-In Services Reference .
Run As User
Integration Server user ID to run the flow service configured to execute when a request violates this rule. Specify Run As User only if the alert type is Flow Service.
