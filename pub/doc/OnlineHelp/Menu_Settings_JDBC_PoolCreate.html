Create JDBC Connection Pool Alias Screen
Screen Heading: Settings > JDBC Pools > Connection Aliases > New
Use the Create JDBC Connection Pool Alias screen to associate a pool of JDBC connections with a database. This database is part of the webMethods audit database.
To open the Create JDBC Connection Pool Alias screen, click JDBC Pools under the Settings menu on the Navigation panel. On the JDBC Pools Screen, click Create a New Pool Alias Definition.
Click Return to JDBC Pool Definitions to return to the previous screen.
Copy From
If you want to use the settings of an existing pool alias, select it from the list. When you select an existing pool alias, Integration Server populates the remaining fields on the Create JDBC Connection Pool Alias screen with the values from the selected pool alias with the following exceptions:
The Alias Name field will contain a prefix of "CopyOf_" to prevent duplicate alias names. You can change the Alias Name as necessary.
The Spy and Snoop check boxes and the Spy Attributes and Snoop Logging Parameters fields are cleared. This ensures that the default values for spy and snoop parameters are set when the alias is saved and that each alias has a unique log file name for the spy and snoop log files.
Alias Name
The alias name that you want to use for the database. You can give the database any alias name you want. The name can be any length and contain any characters.
Alias Description
A brief description of the pool alias.
Associated Driver Alias
The alias of a database driver you want to use for this database. Driver aliases are defined on the Settings > JDBC Pools > Driver Aliases screen.
Database URL
The URL for the database, for example,
jdbc:wm:sqlserver://hostname:port;DatabaseName=AuditDB;SelectMethod=Cursor
Sample Database URL
A sample URL format for the database driver. The sample format displayed depends on the alias you selected in the Associated Driver Alias drop-down above. You can copy this sample, paste it into the Database URL field, and modify it as necessary.
Spy
When selected, enables the DataDirect Spy diagnostic feature for DataDirect Connect JDBC drivers. DataDirect Spy logs JDBC calls and SQL statement interactions between Integration Server and an external RDBMS.
This check box is cleared by default.
Note:  
This option is for use with an external RDBMS only. It is not for use with the embedded IS internal database.
Spy Attributes
The name and location of the log file where Integration Server will log diagnostic data collected by the DataDirect Spy diagnostic feature. This value also defines DataDirect Spy attributes. If this field is empty, Integration Server uses the value specified by the watt.server.jdbc.datadirect.spy.default server configuration parameter.
The default value for watt.server.jdbc.datadirect.spy.default is:
SpyAttributes=(log=(file)/logs/spy/<alias_name>.log;logTName=yes;timestamp=yes)
Where <alias_name> is the name of the JDBC connection pool alias.
Typically, you do not need to change the default value. However, if the attributes do not meet the needs of your system, enter a new value in the Spy Attributes field.
Note:  
If you specify your own value, be aware that the diagnostic tool collects data from the Integration Server_directory /instances/instance_name/logs/spy directory. If you change the log file location, the diagnostic utility might not import the data logged by DataDirect Spy. For more information about this property, see the webMethods Integration Server Administrator’s Guide . For more information about setting DataDirect Spy attributes, consult the documentation on the DataDirect website.
Snoop
When selected, Integration Server enables the DataDirect Snoop tool for DataDirect Connect JDBC drivers. The DataDirect Snoop tool logs network packets between Integration Server and an external RDBMS. You can use the resulting log file for tracing and diagnostic purposes.
This check box is cleared by default.
Note:  
This option is for use with an external RDBMS only. It is not for use with the embedded IS internal database.
Snoop Logging Parameters
Defines the name and location of the log file where Integration Server will log diagnostic data collected by the DataDirect Snoop tool. This parameter also defines DataDirect Snoop tool attributes. If this field is empty, Integration Server uses the value specified by the watt.server.jdbc.datadirect.snoop.default server configuration parameter.
The default value for watt.server.jdbc.datadirect.snoop.default is:
ddtdbg.ProtocolTraceEnable=true;ddtdbg.ProtocolTraceMaxline=16;ddtdbg.ProtocolTraceLocation=/logs/snoop/alias_name.log;ddtdbg.ProtocolTraceShowTime=true
Where alias_name is the name of the JDBC connection pool alias.
Note:  
For DB2, you must include the following command at the end of the value:
ddtdbg.ProtocolTraceEBCDIC=true
Typically, you do not need to change the default value. However, if the attributes do not meet the needs of your system, enter a new value in the Snoop Logging Parameters field.
Note:  
If you specify your own value, be aware that the diagnostic tool collects data from the Integration Server_directory /instances/instance_name/logs/snoop directory. If you change the log file location, the diagnostic utility might not import the data logged by the DataDirect Snoop tool. For more information about this property, see the webMethods Integration Server Administrator’s Guide . For more information about setting the DataDirect Snoop tool attributes, consult the documentation on the DataDirect website.
User Id
The user name that the server must supply to log into the specified database. If a user name and password are not required, leave this field blank.
Password
The password that the server must supply to log into the selected database. If a user name and password are not required, leave this field blank.
Minimum Connections
The minimum number of connections to maintain in the JDBC connection pool. The server creates these connections when it first receives a request to connect to the database. If more connections are needed, the server creates additional connections until the value for Maximum Connections is reached. The default is 0.
Maximum Connections
The maximum number of connections to maintain in the JDBC connection pool. When the number of connection requests reaches this value, the server blocks the requests. The default is 1.
Available Connections Warning Threshold
The number of connections (as a percentage) available in the JDBC connection pool. The connection threshold is calculated using the Maximum Connections multiplied by the specified percentage. When the percentage of available connections in the JDBC connection pool falls to a value that is equal to or smaller than this threshold number, Integration Server logs a message to the server log. If, after the threshold has been exceeded, the number of available connections increases to a value greater than the value of this threshold, Integration Server logs a message stating that the connection pool threshold has been cleared.
The default is 0, meaning that connection threshold checking is disabled.
Waiting Thread Threshold Count
The number of outstanding requests for JDBC connections to allow. When this threshold is exceeded, Integration Server logs a message to the server log and starts a 5 minute interval timer. At the end of the interval, if the number of threads waiting for a connection still exceeds the value of this parameter, Integration Server logs a message to the server log. The default is 0, meaning that waiting thread threshold checking is disabled.
Idle Timeout
The amount of time the server waits before discarding an inactive connection from the JDBC connection pool.
The server only discards connections if the pool contains more than the minimum number of connections. Afterwards, if more connections are needed than exist in the pool, the server creates new ones. The default expiration time is 60000 milliseconds.
Test Connection
Validates the connection to the database, using the connection information defined in the pool alias.
Save Settings
Applies any changes made to the settings.
