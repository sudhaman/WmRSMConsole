Mobile Application Protection Options Screen
Screen Heading: Security > Enterprise Gateway Rules > Mobile Application Protection Options
Use the Mobile Application Protection Options screen to specify mobile device types and mobile applications from which you want to restrict requests. For example, you can deny requests from out-of-date versions of BigApp running on a ZZZ smart phone. Device types and mobile applications listed on this screen are available for you to specify in a mobile application protection filter in an Enterprise Gateway rule.
Important!  
Enterprise Gateway runs on an Integration Server that has been configured as an Enterprise Gateway Server running in your DMZ. The Enterprise Gateway Server requires a webMethods Enterprise Gateway license.
To open the Mobile Application Protection Options screen, click Enterprise Gateway Rules under the Security menu on the Navigation panel, and then click Mobile Application Protection Options.
Click Edit Device Types to specify device types from which you want to restrict requests.
Click Edit Mobile Applications to specify mobile applications from which you want to restrict requests.
Device Types
Device types that you can specify in a mobile application protection filter. With the filter in place, Enterprise Gateway Server will deny or send alerts about requests received from the specified device types.
Mobile Applications
Mobile applications that you can specify in a mobile application protection filter. With the filter in place, Enterprise Gateway Server will deny or send alerts about requests received from the specified mobile applications.
