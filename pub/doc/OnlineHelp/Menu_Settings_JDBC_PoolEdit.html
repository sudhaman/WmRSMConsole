Edit JDBC Connection Pool Alias Screen
Screen Heading: Settings > JDBC Pools > Connection Aliases > Edit
Use the Edit JDBC Connection Pool Alias screen to define a pool alias for a database. The pool alias defines the database's location and connection parameters. The server uses this database to collect data for one of its product functions.
The server uses product functions to collect data needed by the webMethods Monitor product. The primary product functions are the ISCoreAudit and ISProcessAudit functions, which together collect data for the webMethods audit database. Other product functions include the Trading Networks function, which points to the Trading Networks database, and the Key Cross Referencing and Echo Suppression function, which the server uses to synchronize updates among various applications and the databases they reference.
To open the Edit JDBC Connection Pool Alias screen, click JDBC Pools under the Settings menu on the Navigation panel. On the JDBC Pools Screen, under Pool Alias Definitions, find the row for the pool alias that you want to access and click Edit in the Edit Pool Alias column.
Click Return to JDBC Pool Definitions to return to the previous screen.
Alias Name
The alias name that you want to use for the database and its connection pool. You can give the alias any name you want. The name can be any length and contain any characters.
Alias Description
A brief description of the pool alias.
Associated Driver Alias
The alias of a database driver you want to use for this database. Driver aliases are defined on the Edit JDBC Connection Driver Alias Screen.
Database URL
The URL for the database, for example,
jdbc:wm:sqlserver://hostname:port;DatabaseName=AuditDB;SelectMethod=Cursor
Sample Database URL
A sample URL format for the database driver. The sample format displayed depends on the alias you selected in the Associated Driver Alias drop-down above. You can copy this sample, paste it into the Database URL field, and modify it as necessary.
Spy
When selected, enables the DataDirect Spy diagnostic feature for DataDirect Connect JDBC drivers. DataDirect Spy logs JDBC calls and SQL statement interactions between Integration Server and an external RDBMS.
This check box is cleared by default.
Note:  
This option is for use with an external RDBMS only. It is not for use with the embedded IS internal database.
Spy Attributes
The name and location of the log file where Integration Server will log diagnostic data collected by the DataDirect Spy diagnostic feature. This value also defines DataDirect Spy attributes. The default value is:
SpyAttributes=(log=(file)/logs/spy/<alias_name>.log;logTName=yes;timestamp=yes)
Where <alias_name> is the name of the JDBC connection pool alias.
Typically, you do not need to change the default value. However, you can modify the value if the attributes do not meet the needs of your system. Be aware that the diagnostic tool collects data from the Integration Server_directory /instances/instance_name/logs/spy directory. If you change the log file location, the diagnostic utility might not import the data logged by DataDirect Spy. For more information about using the diagnostic utility, see webMethods Integration Server Administratorâ€™s Guide . For more information about setting DataDirect Spy attributes, consult the documentation on the DataDirect website.
Snoop
When selected, Integration Server enables the DataDirect Snoop tool for DataDirect Connect JDBC drivers. The DataDirect Snoop tool logs network packets between Integration Server and an external RDBMS. You can use the resulting log file for tracing and diagnostic purposes.
This check box is cleared by default.
Note:  
This option is for use with an external RDBMS only. It is not for use with the embedded IS internal database.
Snoop Logging Parameters
Defines the name and location of the log file where Integration Server will log diagnostic data collected by the DataDirect Snoop tool. This value also defines DataDirect Snoop tool attributes. The default value is:
ddtdbg.ProtocolTraceEnable=true;ddtdbg.ProtocolTraceMaxline=16;ddtdbg.ProtocolTraceLocation=/logs/snoop/<alias_name>.log;ddtdbg.ProtocolTraceShowTime=true
Where <alias_name> is the name of the JDBC connection pool alias.
Note:  
For DB2, include the following command at the end of the value:
ddtdbg.ProtocolTraceEBCDIC=true
Typically, you do not need to change the default value. However, you can modify the value if the attributes do not meet the needs of your system. Be aware that the diagnostic tool collects data from the Integration Server_directory /instances/instance_name/logs/snoop directory. If you change the log file location, the diagnostic utility might not import the data logged by the DataDirect Snoop tool. For more information about using the diagnostic utility, see webMethods Integration Server Administratorâ€™s Guide . For more information about setting the DataDirect Snoop tool attributes, consult the documentation on the DataDirect website.
User ID
The user name that the server must supply to log into the specified database. If a user name and password are not required, leave this field blank.
Password
The password that the server must supply to log into the selected database. If a user name and password are not required, leave this field blank.
Minimum Connections
The minimum number of connections to maintain in the JDBC connection pool associated with the database. The server creates these connections when it first receives a request to connect to the database. If more connections are needed, the server creates additional connections until the value for Maximum Connections is reached. The default is 0.
Maximum Connections
The maximum number of connections to maintain in the JDBC connection pool associated with the database. When the number of connection requests reaches this value, the server blocks the requests. The default is 1.
Available Connections Warning Threshold
The number of connections (as a percentage) available in the JDBC connection pool. The connection threshold is calculated using the Maximum Connections multiplied by the specified percentage. When the percentage of available connections in the JDBC connection pool falls to a value that is equal to or smaller than this threshold number, Integration Server logs a message to the server log. If, after the threshold has been exceeded, the number of available connections increases to a value greater than the value of this threshold, Integration Server logs a message stating that the connection pool threshold has been cleared.
The default is 0, meaning that connection threshold checking is disabled.
Waiting Thread Threshold Count
The number of outstanding requests for JDBC connections to allow. When this threshold is exceeded, Integration Server logs a message to the server log and starts a 5 minute interval timer. At the end of the interval, if the number of threads waiting for a connection still exceeds the value of this parameter, Integration Server logs a message to the server log. The default is 0, meaning that waiting thread threshold checking is disabled.
Idle Timeout
The amount of time the server waits before discarding an inactive connection from the JDBC connection pool. The server only discards connections if the pool contains more than the minimum number of connections. Afterwards, if more connections are needed than exist in the pool, the server creates new ones. The default expiration time is 60000 milliseconds.
Test Connection
Validates the connection to the database, using the connection information defined in the pool alias.
Save Settings
Applies any changes made to the settings.
