Synchronize Cache Settings from Terracotta Server Array Screen
Screen Heading: Settings > Caching > cacheManagerName > cacheName > Synchronize
Use the Synchronize Cache Settings from Terracotta Server Array screen to synchronize the cache settings if differences exist between cache-wide parameter settings stored in the configuration file on your Integration Server and settings that are actually in effect for the cache (as specified on the Terracotta Server Array). The cache-wide parameters are: Maximum Elements in Memory, Eternal, Time to Live, Time to Idle, Maximum Entries Local Disk, and Logging.
To open the Synchronize Cache Settings from Terracotta Server Array screen, click Caching in the Settings menu of the Navigation panel. In the cache managers list, click the name of the cache manager associated with the cache you want to synchronize. Click the cache name link under Cache Name. The Edit Cache screen displays the Synchronize Cache Settings from Terracotta Server Array link if differences exist between cache-wide settings stored in the local configuration file and the settings that are in effect for the cache. Click Synchronize Cache Settings from Terracotta Server Array.
Click Return to Cache Manager Name to return to the Cache Manager Details screen.
Cache Configuration
Displays the Integration Server and Terracotta Server Array cache settings. The differences between the two sets of cache settings are highlighted in red text.
BigMemory
These parameters are applicable to distributed caches.
Distributed Cache Configuration
Displays the settings from the configuration file on the Integration Server and the settings that are currently in effect for the cache.
Synchronize
Click Synchronize to copy the cache-wide settings from the Terracotta Server Array to the local configuration file on Integration Server.
