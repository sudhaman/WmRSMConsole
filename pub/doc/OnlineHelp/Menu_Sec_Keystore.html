Keystore Screen
Screen Heading: Security > Keystore
Use the Keystore screen to view and provide support for the keystores and truststores used by Integration Server. A keystore contains private keys and certificates for an Integration Server, partner application, or Integration Server component. A truststore contains the trusted root of the certificate or signing authority (CA). From this screen you can also create aliases for the keystores and truststore, so they can be easily referenced from within Integration Server.
Important!  
webMethods does not provide tools for the creation and management of keystores and truststores; they must be created elsewhere and imported into Integration Server.
To open the Keystore screen, click Keystore under the Security menu on the Navigation panel.
Integration Server creates default keystore and truststore aliases named DEFAULT_IS_KEYSTORE and DEFAULT_IS_TRUSTSTORE respectively the first time Integration Server starts. The DEFAULT_IS_KEYSTORE points to the Integration Server_directory /common/conf/keystore.jks file and the DEFAULT_IS_TRUSTSTORE points to the C:\SoftwareAG\common\conf\platform_truststore.jks file. The default password is “manage” for all the entries.
Important!  
Use these default aliases only in development environments where testing and debugging is performed. Do not use these aliases in a production environment.
Click Create Keystore Alias to configure a new alias for an existing keystore.
Click Create Truststore Alias to configure a new alias for an existing truststore.
Click Add Security Provider to add a new security provider, which will make additional keystore or truststore file types available for Integration Server to use.
Keystore/Truststore List
Alias
Displays the user-specified identifier for the keystore or truststore. Clicking on the keystore or truststore alias displays the properties for the alias.
Type
Displays the file format of the keystore or truststore file. The default is JKS for keystores and truststores. You can also use PKCS12 for keystores. Other keystore and truststore types can be made available by loading additional security providers, or by setting the following server configuration properties: watt.security.keyStoreTypes (for keystores) and watt.security.TrustStoreTypes (for truststores).
Provider
Displays the provider that is used for the keystore or truststore type. The default is the provider shipped with the JVM (Oracle, IBM, or others). If your Hardware Security Module (HSM) device is not supported by the default provider, you can configure a different provider to support keystore or truststore file types.
Note:  
Integration Server supports both PKCS12 and JKS file formats for keystores, but only supports JKS for truststores.
Location
Displays the location of the keystore or truststore file on the server. This can be the full pathname, or a relative path in relation to the Integration Server.
Loaded
Indicates whether the keystore or truststore file has been loaded into the Integration Server, and can be used for authentication.
Reload
Click to reload the keystore or truststore file. You may need to reload if changes were made to the security settings since the server was last started, or in case of system outage.
Delete
Click to delete a keystore or truststore alias from the list.
