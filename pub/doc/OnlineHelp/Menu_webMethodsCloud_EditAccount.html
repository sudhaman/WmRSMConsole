Edit Account Screen
Screen Heading:webMethods Cloud > Accounts > Edit Account
Use the Edit Account screen to edit existing webMethods Cloud accounts.
To open the Edit Account screen, click the name of the account you want to edit from the Alias column of the On-Premise Accounts area on the webMethods Cloud > Accounts screen.
Click Return to On-Premise Accounts to return to the list of accounts.
General Settings
Alias Name
A unique name for the account. This field cannot be changed.
Description
Description of the account.
Stage
The webMethods Cloud stage from which the on-premise Integration Server listens for messages. The list is populated by the stages defined in webMethods Cloud. This field cannot be changed.
Account Settings
Maximum Reconnection Attempts
Specify the maximum number of reconnection attempts that Integration Server should make if the connection to webMethods Cloud fails. If the connection cannot be re-established, Integration Server writes messages to the error log and the account will be disabled. The default is 5 attempts.
Time to Live
Maximum amount of time (in milliseconds) that webMethods Cloud waits for the on-premise Integration Server to process a message. If the on-premise Integration Server is not listening for a message or if it takes longer to process the message than the specified time, webMethods Cloud issues an error. A value of 0 (the default) indicates that messages do not have a time-to-live expiration.
Allowed On-Premise Hosts
(Optional.) The on-premise Integration Server might use multiple addresses, depending on which network or proxy it uses to access the webMethods Cloud. Specify a comma-separated list of IP addresses that can receive messages from webMethods Cloud. If no value is specified, webMethods Cloud derives the value of the on-premise Integration Server that uploads the account to webMethods Cloud.
Run As User
The user name you want the on-premise Integration Server to use when running the service. Click to search for and select the user. You can select users from the local or central directory.
The on-premise Integration Server runs the service as if the user you specify is the authenticated user that invoked the service. If the service is governed by an ACL, be sure to specify a user that is allowed to invoke the service.
Test Account Settings
Click to test the account settings. Integration Server Administrator displays a status line that indicates whether the account is valid or not. The status line is displayed at the top of the screen.
Save Changes
Click to save the account.
