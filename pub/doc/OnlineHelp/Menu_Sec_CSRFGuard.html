CSRF Guard Settings Screen
Screen Heading: Security > CSRF Guard
The CSRF Guard Settings screen displays the CSRF guard settings in Integration Server.
Integration Server uses the CSRF guard feature to prevent CSRF attacks. Integration Server applies CSRF guard by creating one CSRF secure token per session and adding it to the subsequent requests until the session expires. Integration Server creates a CSRF secure token when you log in. This token expires when the session ends.
When you send a request, Integration Server verifies the existence and validity of the token in the request and compare it to the token in the session. If the token is not found in the request or if the token in the request does not match the token in the session, Integration Server terminates the request. Integration Server also logs the event as a potential CSRF attack in the server log and the security audit log.
To open the CSRF Guard Settings screen, click CSRF Guard under the Security menu on the Navigation panel.
Click Edit CSRF Guard Settings to enable or disable CSRF guard and to configure the CSRF guard settings.
Enabled
Specifies whether CSRF guard is enabled in Integration Server. True indicates that CSRF guard is enabled. False indicates that CSRF is not enabled. CSRF guard is disabled by default.
Excluded User Agents
Displays the list of user agents for which Integration Server does not apply CSRF guard. If CSRF guard is enabled, Integration Server requires that HTTP requests coming from user agents that are not specified in this field must contain CSRF secure tokens.
Landing Pages
Displays the list of landing pages for the packages in your Integration Server. A landing page is the home page for a package. Integration Server does not check for CSRF secure tokens in the landing pages, but inserts a token for that page. Integration Server guards all further requests from these landing pages with CSRF secure tokens.
Unprotected URLs
Displays the URLs for which Integration Server does not check for CSRF secure tokens. Integration Server requires that the requests coming from all URLs that are not specified in this field must contain CSRF secure tokens.
If you specify a DSP file as an unprotected URL, Integration Server will not insert a CSRF secure token for that file. If you attempt to access any other protected page from this DSP page, Integration Server issues an error or redirects you to the home page of Integration Server Administrator depending on your Denial Action configuration.
Denial Action
Displays the action that you have configured Integration Server to perform when it detects that a request does not contain a CSRF secure token or contains an invalid CSRF secure token.
Error indicates that you want Integration Server to throw an access denied error and terminate the request. This is the default.
Redirect specifies that Integration Server is to redirect the user to a confirmation page or the home page of Integration Server Administrator.
