Edit Logging Settings Screen
Related Topics
Screen Heading: Settings > Logging > Edit
Use the Edit Logging Settings screen to edit the settings that control the way in which the server logs operating and error information.
To open the Edit Logging Settings screen, open the Logging Screen and click Edit Logging Settings.
n
Click Return to Logging Settings to return to the Logging Settings screen.
Logging
Use the fields in this section to specify the type of information that you want the server to record in its log files.
Level of Logging
Specifies the level of detail that the server records in the Server Log. Specify a value from 1 to 10, where 10 produces the highest level of detail in the log file.
 
  
Level
Description
 
Off
No messages.
 
Fatal
Fatal messages only.
 
Error
Error and fatal messages.
 
Warn
Warning, error, and fatal messages.
 
Info
Informational, warning, error, and fatal messages. This is the default.
 
Debug
Debug, informational, warning, error, and fatal messages.
 
Trace
Trace, debug, informational, warning, error, and fatal messages.
Facilities
Specifies the facilities whose activities you want the server to log.
Email Notification
Use this section to specify who is notified via email when certain events occur on the server.
SMTP Server
Specifies the address of the Simple Mail Transfer Protocol (SMTP) server through which the server will issue e-mail notifications.
Internal Email
Optional. Specifies the e-mail address to which the server sends a message when an internal server error occurs (usually the server's administrator). E-mail address of the person to receive administrator messages, such as warnings about the server key expiring or excessive failed login attempts. Typically, this is the e-mail address of the administrator for the Integration Server.
Service Email
Optional. Specifies the e-mail address to which the server sends a message when an output binding fails (usually a developer who is familiar with the services on your server). The e-mail address of the person to receive messages about application errors. During the development phase, you might direct exceptions to the developer to aid debugging. Once in production, you might direct exceptions to the application user.
Default Email Charset
Character format the server uses for e-mails it sends. The default is utf8.
Log Timestamp Format
Specifies the format in which the server displays the date and time in its log files. Set the format to one that is supported by the Java class java.txt.SimpleDateFormat, for example yyyy-MM-dd HH:mm:ss.
