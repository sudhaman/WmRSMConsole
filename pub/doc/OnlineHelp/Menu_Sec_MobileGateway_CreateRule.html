Create Rule Screen
Screen Heading: Security > Mobile Gateway > Create Rule
Use the Create Rule screen to configure the Mobile Gateway rule properties. You can specify the action that you want Integration Server to perform when a request violates a rule and the filters that you want Integration Server to apply on incoming requests.
To open the Create Rule screen, click Mobile Gateway under the Security menu on the Navigation panel. Click Create Rule.
Click Return to Mobile Gateway to return to the Mobile Gateway page.
Rule Properties
Rule Name
Enter a name for the Mobile Gateway rule. Valid rule names:
Must be unique.
Must not be null.
Must not contain spaces.
Must not contain these characters:
? ~ ` ! @ # $ % ^ & * ( ) - + = { } | [ ] \\ : \" ; ' < > , /
Description
Enter a brief description about the Mobile Gateway rule.
Enable
Select the Enabled check box if you want to enable the rule immediately. Integration Server applies a rule to requests only if the rule is enabled. You can enable the rule later from the Security > Mobile Gateway page.
Applies to
Service Type
Select the type of request to which you want to apply the Mobile Gateway rule. Select All if you want to apply this rule to all requests. Select SOAP, REST, or INVOKE to apply this rule to SOAP, REST, or INVOKE services respectively.
Service Name
Specify a string to filter the services based on its name. You can specify the service name as a regular expression. The asterisk (*) is the only wildcard character allowed in the regular expression.
Note: 
You can specify a service name only if you selected a service type from the Service Type list.
Action
Select one of the following:
Select...
To...
Deny Request and Alert
Deny the request if it violates the rule. Integration Server sends an alert based on the configured alert options.
Alert
Allow the request and send an alert based on the configured alert options.
Error Message
Enter a custom message that you want Integration Server to send in case a request violates a Mobile Gateway rule.
Filters
Message Size Limit
Select the check box to specify the maximum message size that this rule enforces.
Maximum Message Size
Enter the maximum size allowed for HTTP and HTTPS requests. If the request is larger than the size specified in this limit, the rule will be violated and Integration Server will perform the configured action.
Maximum Chunk Size
Enter the maximum chunk size allowed for Transfer-Encoding:Chunked requests. If the chunk is larger than the size specified in this limit, the rule will be violated and Integration Server will perform the configured action.
Save Changes
Saves the Mobile Gateway rule.
Integration Server displays the rule in the Mobile Gateway page under Denial Rules or Alert Rules depending on the action you selected when you created the rule.
