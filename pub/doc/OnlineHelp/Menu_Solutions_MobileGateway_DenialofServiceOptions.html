Denial of Service Options Screen
Screen Heading: Mobile Gateway > Denial of Service Options
Use the Denial of Service Options screen to limit the number of requests a Reverse HTTP Gateway Server will accept and the number it will process concurrently. By limiting requests this way you can prevent Denial of Service (DoS) attacks that flood Integration Server with requests in an attempt to interfere with Integration Server processing.
Important! 
Mobile Gateway runs on an Integration Server that has been configured as a Reverse HTTP Gateway Server running in your DMZ. The Reverse HTTP Gateway Server requires a Reverse Gateway license. Mobile Gateway requires a separate Mobile Gateway license.
To open the Denial of Service Options screen, click Mobile Gateway under the Solutions menu on the Navigation panel, and then click Denial of Service Options.
Click Global Denial of Service to configure global limits on requests. When global settings are in effect, Integration Server considers all requests, regardless of the IP address from which a request was sent.
Click Denial of Service by IP Address to configure limits on requests based on the IP address from which a request was sent.
Global Denial of Service
Enabled
Whether Integration Server is enforcing limits on the total number of requests it will accept and the total number it will process concurrently, regardless of IP address from which a request was sent.
Maximum Requests
Maximum number of requests Integration Server will accept in the specified time interval (in seconds), regardless of the IP address from which a request was sent.
Maximum Requests in Progress
Maximum number of requests Integration Server will run concurrently. Integration Server considers all requests, regardless of the IP address from which the request was sent.
Block Interval
Length of time (in minutes) that Integration Server is to block client requests from this IP address when the number of requests exceeds the value specified in the Maximum Requests or Maximum Requests in Progress fields. Integration Server blocks client requests from this IP address at every Gateway External port on the Reverse HTTP Gateway Server.
Denial of Service by IP Address
Enabled
Whether Integration Server is to enforce limits on the number of requests it will accept or the number of requests it will process concurrently from any individual IP address. For example, you can specify that if Integration Server receives more than 15 requests from any individual IP address in 10 seconds that subsequent requests from that IP address will be denied.
Maximum Requests
Maximum number of requests Integration Server will accept from any individual IP address in the specified time interval.
Maximum Requests in Progress
Maximum number of requests Integration Server will run concurrently from the same IP address.
When limit exceeds
Action to take if the limit specified by the Maximum Requests or Maximum Requests in Progress field is exceeded.
Select...
To...
Add to Deny List
Permanently add the IP address that exceeded the limit to the Mobile Gateway deny list. With this setting in effect, Integration Server blocks requests from the IP address at every Gateway External port on the HTTP Reverse Gateway Server. The address remains in the Mobile Gateway deny list until an administrator deletes it from the list.
Block
Temporarily add the IP address that exceeded the limit to the Mobile Gateway deny list. With this setting in effect, Integration Server blocks requests from the IP address at every Gateway External port on the HTTP Reverse Gateway Server for a period of time that you specify.
Block Interval
Length of time (in minutes) that Integration Server is to block requests from an IP address when the number of requests exceeds the value specified in the Maximum Requests or Maximum Requests in Progress field. Integration Server blocks client requests from this IP address at every Gateway External port on the Reverse HTTP Gateway Server.
Denied IP Address List
IP Address
List of IP addresses that have been blocked or added to the Mobile Gateway deny list for sending requests that exceed configured limits for the number of requests received or the number of requests to process concurrently. When the requested action is Block, an IP address is displayed in the list until it is automatically deleted after the block interval has passed. If the requested action is Add to Deny List, the IP address remains in the list until an administrator deletes it.
Important! 
The Mobile Gateway deny list takes precedence over the allow/deny list for the Gateway External Server port.
Delete
Deletes the IP address from the deny list. After you delete the IP address, Mobile Gateway will allow requests from this IP address again.
