Reset All Outbound Passwords Screen
Screen Heading: Security > Outbound Passwords > Reset All Outbound Passwords
If you lose or forget the master password or it becomes corrupted, use the Reset All Outbound Passwords screen to reset all the stored outbound passwords and the master password. This procedure clears (blanks out) the stored outbound passwords and resets the master password to "manage." In addition, you must manually re-enter the appropriate passwords for all application and subsystem passwords on their respective configuration screens in the Integration Server Administrator.
Important!  
Reset the outbound passwords only if your Integration Server encrypts outbound passwords using PBE and you have lost your master password. Once you reset the passwords, the Integration Server will no longer be able to connect to the applications and subsystems for which the passwords are required, until you manually reset them.
To open the Reset All Outbound Passwords screen, click Outbound Passwords under the Security menu on the Navigation panel. Click Update Master Password. Click Reset All Outbound Passwords.
Reset Passwords
Resets the master password to "manage" and deletes all outbound passwords. Once you have reset the passwords, you must do the following:
1.
From the Update Master Password Screen change the master password something other than “manage.”
2.
Go to the configuration screen for each application or subsystem and re-enter the password required for the Integration Server to connect to that application or subsystem. Screens to check include those that define:
Remote Integration Servers
Clustered servers
JDBC connection pools
POP/IMAP servers
LDAP servers
Proxy servers
Internal Server Port
Email Port
Databases
Keystores
Web Services Endpoint Aliases
