Guaranteed Delivery - Outbound Log Screen
Screen Heading: Logs > Guaranteed Delivery > Outbound
Use the Guaranteed Delivery Outbound Log screen to view the outbound transaction log. The outbound transaction log (WMTXOUT_yyyymmdd_hhmmss.log) records information about guaranteed-delivery requests that the server makes to other webMethods Integration Servers. The log contains a record of all critical operations that the server performs for each guaranteed delivery transaction.
You can use these logs with the appropriate job store to identify specific information about a transaction. For example, if a transaction fails, you can use the log to determine how far the transaction got before failing so you can take appropriate measures to recover it.
For information about changing the display of this log, see webMethods Audit Logging Guide .
To open the Guaranteed Delivery Outbound Log screen, click Guaranteed Delivery under the Logs menu on the Navigation panel. From the Guaranteed Delivery - Inbound Log Screen, click View Guaranteed Delivery Outbound Log.
Click View Guaranteed Delivery Inbound Log from the Outbound Guaranteed Delivery log to see the inbound log. The inbound transaction log (WMTXIN_yyyymmdd_hhmmss.log) records information about guaranteed-delivery requests that the server processes.
Log display controls
Display Log Entries oldest to the newest starting from the beginning
View entries in the log from the oldest to the most current.
Display Log Entries newest to the oldest starting from the end
View entries in the log from the most current to the oldest.
Number of entries to display
By default, the Integration Server Administrator displays the 35 most recent entries in the log. You can change the number of entries displayed.
Tip!  
If you have Administrator privileges, you can permanently change the default value of this field. Go to the Settings > Extended page and modify the watt.server.log.maxEntries parameter.
From/To
Specify, in YYYY-MM-DD HH:MM:SS date format, the range of dates and times for which you want log entries to be displayed.
Note:  
The From/To fields are displayed only if the logger associated with this log is configured to write to a database. For more information, see the webMethods Audit Logging Guide .
Refresh
Click this button to update the display after changing the log display controls.
Tip!  
If you have Administrator privileges, you can change the default refresh interval for log entries (the default is 90 seconds). Go to the Settings > Extended page and modify the watt.server.log.refreshInterval.
Guaranteed Delivery Log Entries as of mm-dd hh:mm:ss
Time Stamp
Date and time the entry was written to the log.
Status
Status of service execution. The status can be either STARTED, ENDED, or FAILED.
Message
The name of the process that is running.
Error Message
Optional error message, describing the type of error.
Root Context, Parent Context, Current Context
Context information that provides a consistent, interpretable audit trail of activity that the webMethods Monitor can use to relate entries within the log.
