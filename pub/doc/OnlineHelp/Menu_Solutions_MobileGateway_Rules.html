Rules Screen
Screen Heading: Mobile Gateway > Rules
The Rules screen displays a list of Mobile Gateway rules. Use the Rules screen to create, view, edit, copy, delete, enable, or disable a rule. You can also use this screen to change a rule’s alert options or priority.
Important! 
Mobile Gateway runs on an Integration Server that has been configured as a Reverse HTTP Gateway Server running in your DMZ. The Reverse HTTP Gateway Server requires a Reverse Gateway license. Mobile Gateway requires a separate Mobile Gateway license.
To open the Rules screen, click Mobile Gateway under the Solutions menu in the Navigation panel, then click Rules.
Click Create Rule to create new rules.
Denial/Alert Rules
The Denial Rules section displays the rules that enforce the Deny Request and Alert action. The Alert Rules section displays the list of rules that enforce the Alert action.
Rule Name
Name of the rule. Click the underlined name of a rule to view or change details about a rule.
Filters
One or more filters that the rule applies to requests. Depending on the type of rule, if a request meets the conditions of the filter, Integration Server either denies the request and sends an alert about the denial or simply sends an alert.
Enabled
Whether the rule is currently enabled or disabled.
Alert Options
Whether the default alert option is applied to the rule. If the rule uses the default alert option, Integration Server displays Default in this column. If the alert option is specific for the rule, Integration Server displays Custom in this column. Click the Default or Custom link to view or edit the alert options.
Copy
Creates a copy of the rule. You can create a new Mobile Gateway rule by copying an existing rule. When you copy a rule, the filter settings, alert options, and other properties, except for rule name, are copied.
Delete
Deletes the rule.
Priority
Enables you to change the order in which Integration Server enforces the rules on requests.
