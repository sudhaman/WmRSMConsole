Default Alert Options Screen
Screen Heading: Mobile Gateway > Default Alert Options
Use the Default Alert Options screen to view the settings for the default alert options for Mobile Gateway rules. Integration Server uses the default alert options when a rule does not specify alert options of its own.
Important! 
Mobile Gateway runs on an Integration Server that has been configured as a Reverse HTTP Gateway Server running in your DMZ. The Reverse HTTP Gateway Server requires a Reverse Gateway license. Mobile Gateway requires a separate Mobile Gateway license.
To open the Default Alert Options screen, click Mobile Gateway under the Solutions menu on the Navigation panel, then click Default Alert Options.
Click Edit Default Alert Options to configure the alerts.
Alert Type
Type of alert Integration Server issues when a request violates a Mobile Gateway rule. Can be None, E-mail, or Flow Service.
Send Alert
Frequency with which Integration Server sends an alert when a rule is violated.
Select...
To...
On Rule Violation
Send an alert each time a request violates the rule.
Every number minutes
Limit the number of alerts sent within a specified time interval. When you select this option, Integration Server sends an alert the first time a request violates a rule. If the violation occurs again within the specified time interval, Integration Server waits until the end of the time interval before sending another alert. For example, suppose you specify the time interval as 1 minute. At 10:00 a request violates the rule and Integration Server sends an alert. Ten seconds later another request violates the rule. Ten seconds after that yet another request violates the rule. Integration Server waits until 10:01 to send a summary of alerts that were generated in the time interval.
E-mail IDs
One or more E-mail IDs to which Integration Server sends an alert when a request violates this rule. Specify e-mail IDs only if the alert type is E-mail. To use this option, the Reverse HTTP Gateway Server on which Mobile Gateway runs must be configured to send e-mails. Check the settings on the Settings > Resources screen under Email Notification. If your Integration Server is not configured to send e-mails, see Administering webMethods Integration Server for instructions on configuring Integration Server to have this ability.
folder.subfolder:service
Name of the flow service to run when a request violates a Mobile Gateway rule. Specify a value in this field only if the alert type is Flow Service. Use the pub.security.mobilegateway:alertSpec specification as the signature for the flow service. For more information about the specification, see webMethods Integration Server Built-In Services Reference.
Run As User
Integration Server user ID to run the flow service that Integration Server will run when a request violates a Mobile Gateway rule. Specify a Run As User only if the alert type is Flow Service.
