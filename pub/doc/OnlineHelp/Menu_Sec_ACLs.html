Access Control Lists (ACLs) Screen
Screen Heading: Security > Access Control Lists
Use the Access Control Lists (ACLs) screen to specify the membership of an ACL. An ACL controls which user groups are permitted to use resources (e.g., services, DSPs, folders) on your server.
To open the ACLs screen, click ACLs under the Security menu on the Navigation panel.
Click Add and Remove ACLs to create a new ACL or delete an existing one.
ACL Membership
Use this section to specify which user groups are explicitly permitted to use, or prohibited from using, a resource governed by an ACL.
Select ACL
Lists the ACLs that are currently defined on your server. To view or edit an ACL, select it from this list.
Group Associations for this ACL
Use this section to specify the membership in the selected ACL. A group can have one of three possible states within an ACL.
State
Description
Allowed
Specifies groups whose members are permitted to use the resources to which this ACL is assigned.
Note:  
If a member in an Allowed group is also a member of one or more Denied groups, that member will be denied access to the resources governed by the ACL.
Denied
The members of these groups are explicitly prohibited from using the resources that are governed by this ACL.
Use the -> and <- buttons to move a selected group to another list. You can move multiple groups by clicking multiple names in a list.
The string "@--@" following a group name indicates that information about the group is stored in the My webMethods Server database and that the database is currently unavailable. Integration Server checks periodically for the availability of this database. When the database becomes available again, Integration Server will remove the "@--@" from the group name. The watt.server.acl.groupScanInternal property controls how often Integration Server checks the database. Resulting Local users in this ACL with access:
Displays the names of users that the ACL authorizes, given the current settings in the Allowed and Denied lists.
Save Changes
Updates the ACL based on the settings specified in ACL Membership.
