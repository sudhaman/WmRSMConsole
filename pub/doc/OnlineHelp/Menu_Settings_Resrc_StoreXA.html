Edit XA Recovery Store Settings Screen
Screen Heading: Settings > Resources > Store Settings > Edit XA Recovery Store Settings
Use the Edit XA Recovery Store Settings screen to configure the location and initial size of the XA recovery store. The XA recovery store saves the XID and the global state of XA transactions. If Integration Server shuts down unexpectedly before the transaction completes, Integration Server can retrieve the state information from the XA recovery store and try to resolve uncompleted transactions, if there are any.
To open the Edit XA Recovery Store Settings screen, click Resources under the Settings menu on the Navigation panel. On the Resources Screen, click Store Settings. On the Store Settings Screen, click Edit XA Recovery Store Settings.
Click Return to Store Settings to return to the previous screen.
XA Recovery Store
Use this section to specify the location and initial size of the XA recovery store.
Store Location
Specify the location of the XA recovery store. By default, the Integration Server saves the XA recovery store in the following directory:

Integration Server_directory
\instances\instance_name\XAStore
If you want to save the XA recovery store in a different location, specify the directory in this field. If the directory does not exist, the server creates it.
Initial Store Size (MB)
Specify the amount of disk space to allocate to the XA recovery store at start up. The XA recovery store automatically increases when it receives data that exceeds the initial size. The default size is 10 MB.
Save Changes
Saves the specified settings. The settings with an asterisk (*) next to them will take effect when you restart the server.
