Mobile Gateway Screen
Screen Heading: Security > Mobile Gateway
The Mobile Gateway screen displays the default alert options as well as the list of Mobile Gateway rules.
Use the Mobile Gateway screen to edit the default alert options and to create, copy, or delete Mobile gateway rules. You can also use this screen to view the filters that the rules enforce, to enable or disable the rules, and to change the priority order of the rules.
Important! 
To use Mobile Gateway, your Integration Server must be equipped with Reverse Gateway and Mobile Gateway licenses from Software AG. Integration Server must also be configured to use the Reverse HTTP Gateway Server.
To open the Mobile Gateway screen, click Mobile Gateway under the Security menu on the Navigation panel.
Click Edit Default Alert Options to configure the alerts.
Click Create Rule to create new rules.
Alert Options
Alert Type
Displays the type of alert that Integration Server issues when a request violates a Mobile Gateway rule.
Send Alert
Displays whether Integration Server issues the alerts whenever a request violates a Mobile Gateway rule or at any specific time interval.
E-mail IDs
Displays the e-mail IDs to which you want Integration Server to send the alerts when a request violates a Mobile Gateway rule. You need to specify e-mail ID only if the alert type is e-mail.
Denial/Alert Rules
Rule Name
Name of the rule. The Denial Rules section displays the rules that enforces the Deny Request and Alert action. Click the underlined name of a rule to view or change details about a rule. The Alert Rules section displays the list of rules that enforce the Alert action.
Filters
Specifies the filters that the rule enforces on requests. If the rule is enabled, Integration Server applies the filters to the requests. If the requests violate any of the filters, Integration Server denies the request and alerts the configured user via e-mail. Integration Server only processes requests that do not violate any of the filter conditions.
Enabled
Specifies whether the rule is currently enabled or disabled. When enabled (Yes), Integration Server enforces the rule on requests. When disabled (No), Integration Server does not enforce the rule. To enable or disable a package, click the No or Yes link in this field.
Alert Options
Specifies whether the alert option applied to the rule is the default alert option or not. If the rule uses the default alert option, Integration Server displays Default in this column. If the alert option is specific to the rule, Integration Server displays Custom in this column. Click the Default or Custom link to edit the alert options.
Copy
Click the icon to create a copy of the rule. You can create a new Mobile Gateway rule by copying an existing rule. When you copy a rule, all the properties and filter settings of the rule gets copied.
Delete
Deletes the rule.
Priority
Specifies the order in which Integration Server enforces the rules on requests.
