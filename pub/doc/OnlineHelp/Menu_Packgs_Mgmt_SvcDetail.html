Service Details Screen
Screen Heading: Packages > Management > packageName > Services > serviceName
Use the Service Details screen to view detailed information about a service or specification.
To open the Service Details screen, go to the Packages Management Screen and click a package name. The Package Description Screen will be displayed. Click Browse services in packageName for a list of the services and specifications in that package. Click a service or specification name to view the details.
Click Return to list of services in package packageName to return to the list of services and specifications in this package.
Click Test serviceName to open the Test Service Screen, which allows you to execute this service.
Click Reset Service Cache to clear all cached results for this service.
General Information
This section displays general information about the service or specification.
Service
Displays the fully qualified name of the service or specification.
Package
Displays the name of the package to which this resource belongs.
Type
Displays the type of resource: Java, Flow, or Specification.
Stateless
Indicates whether the service runs as a "stateless" service. Stateless services are ones for which the server does not maintain "state" information (e.g., information about the requester). Stateless services run faster and use fewer resources. To change the Stateless setting for a service, you use Software AG Designer.
Universal Name
This section displays the universal name of the service, used to reference a service on a webMethods Integration Server.
Namespace Name
Specifies the part of the universal name that qualifies the local name of a service. It must be a valid absolute URI.
Local Name
Specifies the part of the universal name that uniquely identifies this service within the collection encompassed by the Namespace name.
Java-Specific Parameters
This section displays information relating to Java services. This section displays only if the selected service is a Java type.
Class
Specifies name of the Java class containing the code for this service.
Method
Specifies the name of the Java method containing the code for this service.
Access Control
This section displays information about security for this resource.
List ACL
Controls who can list the resource on screens in Designer and Integration Server Administrator. List access also allows a user to view a resource's metadata. Angle brackets <> around an ACL name indicate that the resource inherits its ACL setting from the parent folder. You can change a resource's List ACL on the Set ACL Screen.
Read ACL
Controls who can view the main source of a resource through Designer and Integration Server Administrator. Angle brackets <> around an ACL name indicate that the resource inherits its ACL setting from the parent folder. You can change a resource's Read ACL on the Set ACL Screen.
Write ACL
Controls who can edit the resource. Angle brackets <> around an ACL name indicate that the resource inherits its ACL setting from the parent folder. You can change a resource's Write ACL on the Set ACL Screen.
Execute ACL
Controls who can execute the resource. This access also gives the user access to files the server serves, such as DSP and .htm files. Angle brackets <> around an ACL name indicate that the resource inherits its ACL setting from the parent folder. You can change a service's Execute ACL on the Set ACL Screen.
Enforce ACL on Internal Invokes
Indicates whether access to this service is controlled on internal invokes (when the service is invoked internally by another service) and external invokes (when the service is requested by an external client). When a service is externally invoked, the server always performs an ACL check to verify that the requester is authorized to invoke that service. However, it does not perform an ACL check on internally invoked services unless the Enforce ACL on Internal Invoke option is enabled on the service being invoked. To change the ACL enforcement option for a service, you use Set ACL Screen or Software AG Designer.
Cache Control
This section displays information about the service's use of cache.
Caching
Indicates whether the server caches the contents of the pipeline after invoking this service. To change the Caching setting, you use Software AG Designer.
Cache Expiration
Indicates the amount of time, in minutes, that the results of this service stay in memory after the server caches them. The server uses this setting to determine when to release a cached result from memory. To change the Cache Expiration setting, you use Software AG Designer.
Prefetch
Indicates whether the server will automatically refresh a cached result for this service when it expires and it has been hit the minimum number of times specified in Prefetch Activate. The server refreshes the result by "prefetching" it, that is, re-executing the service with the same set of inputs associated with that entry. To change the Prefetch setting, you use Software AG Designer.
Note:  
The cache may not be refreshed at the exact time specified in Cache Expiration and Prefetch Activate. It may vary from 0 to 15 seconds, according to the cache sweeper thread. Select watt.server.cache.flushMins setting on the Show and Hide Keys Screen to enable the display of the actual cache refresh time on the Extended Settings Screen.
Prefetch Activate
Indicates the minimum number of times that a cached result must be accessed before the server automatically prefetches its results. If a cached result receives at least this many hits before it expires, the server will automatically re-execute the service (with the same set of inputs) to re-cache its results. If a cached result does not receive the minimum number of hits, the server releases it from memory when it expires. To change the Prefetch Activate setting, you use Software AG Designer.
Data Formatting
This section contains information about how the service receives HTML or XML data and how it formats HTML and XML output for a web browser.
Binding Name
Displays the name of the service that parses the incoming HTML or XML for the service. This service is used by services that receive input values from HTML- or XML- documents that are posted to the server. The server uses the binding to locate and extract values from the posted text.
Output Template
Displays the name of the output template assigned to this service. An output template is a set of instructions that defines which output values are returned to an HTTP client and how those values are formatted. To assign an output template to a service, you use Software AG Designer.
Template Type
Indicates the type of output template this service uses. The template type determines how the server sets the Content-Type header field when it returns the results of this service to an HTTP client. To set the template type, you useSoftware AG Designer.
