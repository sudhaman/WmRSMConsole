Configure Client Certificates Screen
Screen Heading: Security > Certificates > Configure Client Certificates
Use the Configure Client Certificates screen to view the list of clients whose certificates are accepted by your server. You also use this screen to add (import) and delete certificates from this list.
Note:  
If your SSL port is configured to "require" client certificates, the port will accept requests from only those clients whose certificates are in this list.
Note:  
When a client connects to the server using a certificate instead of a user name and password, that client is logged on to the server under the user name that is mapped to that particular certificate. You map a user name to a certificate when you import it into this list.
To open this screen, go to the Certificates Screen and click Configure Client Certificates.
Click Return to Certificates to return to the list of current certificate settings.
Import Certificate
Use this section to import a client certificate and map it to a user name.
Certificate Path
Specifies the name of the file containing the certificate that you want to import. You may specify the file name using an absolute path or using a path that is relative to Integration Server_directory . The file must contain an X.509 certificate in DER file format.
User
Specifies a pre-defined user name or provides the ability to choose a user by clicking on the magnifying glass icon next to this field. From this,
Choose Local (CDS) or LDAP as a Provider from the drop-down list.
Next, choose a User from the default list given or Search by name and click Go.
Usage
Specifies the purpose for which you want to import this certificate. The options are SSL Authentication, Verify, Encrypt, Verify and Encrypt, and Message Authentication.
Import Certificate
Imports the certificate specified.
If you import a certificate, a link to the Client Certificates Detail Screen appears.
Current Certificates
Displays a list of certificates that are mapped to users on Integration Server.
Subject CN
Specifies the common name of the organization to which the certificate was sent. If the certificate does not contain a subject field, this field displays (none). Click the value in this field to view details about this certificate.
Issuer CN
Specifies the common name of the organization that issued the certificate.
Serial Number
Specifies a number that uniquely identifies this certificate.
User
Specifies the name of the Integration Server user to which this certificate is mapped.
Usage
Specifies the purpose for which the certificate was imported.
Remove
Removes the mapping for this certificate.
