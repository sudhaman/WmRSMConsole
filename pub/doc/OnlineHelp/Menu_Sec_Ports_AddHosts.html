Add Hosts to Allow/Deny List Screen
Screen Heading: Security > Ports > IP Access > Add/Deny Hosts > Global
Use the Add Hosts screen to specify which IP addresses are explicitly permitted to access a port or explicitly denied access.
To open the Add Hosts screen, go to the Ports Screen and click Change Global IP Access Restrictions. Either Add Hosts to Deny List or Add Hosts to Allow List will be displayed, depending on whether the IP access is Allow by Default or Deny by Default.
Hosts
Specifies the host names (for example, workstation5.webmethods.com) or IP addresses (for example, 132.906.19.22) of hosts from which the server is to accept inbound requests. Separate your entries with commas, for example: *.allowme.com, *.allowme2.com.
Note:  
IP addresses are harder to spoof, and therefore more secure.
The host names or IP addresses can include upper and lower case alphabetic characters, digits (0-9), hyphens (-), and periods (.) and cannot include spaces.
Note:  
IP addresses are harder to spoof, and therefore more secure.
You can use the following pattern-matching characters to identify several clients with similar host names or IP addresses.
Char
Description
Example
*
Matches any number of characters
r*.webMethods.com
?
Matches any single character
workstation?.webMethods.com
